# 2026-06-02

## 한 줄 요약

v2(월별 타임머신)에서 **v3(인생 연혁)** 으로의 메인 UX 피벗을 한 세션에 L1~L7 7단계로 완성. 매달 빈 칸 채우는 부담 → "내 인생 큰 줄기 한 번 잡기" 로 전환하면서 **v2 코드 전체 보존** (사이드 패널/출석/진척/가족 반응/v2 비서/타임머신 월 화면) — 진입 동선만 갈아끼움.

## 0. 피벗 배경

v2 실사용 통찰: **"월마다 말을 남기는 게 너무 부담스럽다."** 빈 칸을 매달 채우라는 건 일기 강박과 같아 어르신껜 더 무거움. 사람은 인생을 "1998년 7월"이 아니라 *"대학 다닐 때"* 식으로 떠올림 — 절대 좌표가 아니라 상대 좌표.

새 방향: **가로 시간축의 인생 연혁**이 메인. 정해진 질문으로 인생 *골격* 잡고, 그 위에 큰 사건들을 사용자가 *원할 때만* 붙임. "매달 채우기"가 아니라 "내 인생의 큰 줄기 한 번 잡기".

원칙:
- v2 코드는 **대체하지 않고 보조로**. 월별 타임머신은 깊이 파고 싶을 때 들어가는 보조 도구로 재배치.
- 새 모델 추가 0. 기존 `UserMemory` 에 `createdVia="life_event"` 디스크리미네이터로 얹어 가족 룸/반응/진척 자동 호환.

## 1. L1 — 데이터 모델 (additive only)

`UserMemory` 에 5 필드 추가:
- `eventTitle: String?` — 인생 이벤트 제목
- `eventYear: Int?` — 정확하든 추정이든 항상 채움
- `eventMonth: Int?` — 모르면 null
- `precision: EventPrecision?` — `EXACT` | `APPROXIMATE`
- `category: LifeCategory?` — 9종 (BIRTH/CHILDHOOD/SCHOOL/MILITARY/WORK/RELATIONSHIP/FAMILY/RESIDENCE/OTHER)

인덱스 `(userId, eventYear, eventMonth)` — 연혁 정렬 쿼리 가속.

**미러링 약속**: `life_event` 행은 `year/month/title` 에도 같은 값을 미러링. 가족 룸의 `listRoomMemories` 가 `year/month/title` 컬럼을 그대로 읽으므로 *코드 0줄 추가로 자동 공유*.

마이그레이션: `20260601153053_add_life_event_fields`. 모든 컬럼 optional → 기존 행 무영향.

검증 (`db/test-life-events.ts`): 다른 사용자 격리, 정렬(연 → 월 NULLS LAST → createdAt), enum 보존, 가족 룸 미러링 확인.

## 2. L2 — 초기 질문 폼 (`/life-record`)

9개 카테고리 질문을 한 번에 한 화면씩, 큰 글씨로. 모두 **건너뛰기 가능**.

`lib/life-record/questions.ts` — 카테고리·짧은 라벨·full 질문·옵셔널 여부·예시 placeholder. `nextUnansweredCategory(answered)` 로 다음 미답 항목 결정.

`/life-record` (인덱스) — 진행도("9개 중 N개"), 다음 미답 카테고리로 가는 큰 버튼, 9개 카드 (답함 ✓ emerald / 아직 zinc, **X 표시 같은 부정 시각 안 씀**). 시니어 친화: 본문 20px+, 카드 64~88px, 4px focus ring.

`/life-record/[category]` — 카테고리별 CategoryForm. EXACT/APPROXIMATE 자동 결정 (month 있으면 EXACT). 다음 미답으로 자동 진행.

`upsertLifeEvent(userId, category, input)` — 카테고리당 최신 1행 upsert. 같은 카테고리 여러 행은 L4 가 만든 자유 추가 — L2 는 가장 최근 1행만 건드림 (L4 데이터 보존).

검증 (`db/test-life-record-flow.ts`): 신규/수정 라운드트립, EXACT↔APPROXIMATE precision, 가족 룸 미러링.

## 3. L3 — 가로 시간축 (`/life-timeline`)

`TimelineView` — events 를 받아 가로 SVG 라인 + 점.
- 앵커(`EXACT`): 진한 큰 점 + 라벨 "1972년 3월 OO초등학교"
- 사이(`APPROXIMATE`): 작은 점/약한 색 + "1985년쯤 대학교 입학"
- 점 클릭 → `/timemachine/[year]/[month]` (month null 이면 `APPROX_DEFAULT_MONTH=6`)

`getLifeEvents(userId)` 정렬:
1. `eventYear ASC` — 인생 흐름
2. `eventMonth ASC NULLS LAST` — 같은 해 안에서 정확이 앞
3. `createdAt ASC` — 같은 (연,월) 안에선 답한 순서 (별도 order 컬럼 안 만듦 — 떠올린 순서 = 머릿속 흐름)

빈 상태: 압박 없는 초대 (🌱 + 큰 violet 버튼 "인생 기록 시작하기 →" → /life-record).

## 4. L4 — 자유 추가/수정/삭제

연혁에서 "+ 인생의 한 장면 추가하기" 버튼 → `/life-timeline/add`. 카테고리당 여러 행 허용 (L2 의 upsert 와 다르게).

`createLifeEvent` / `updateLifeEvent` / `deleteLifeEvent` — 모두 `userId + createdVia="life_event"` 강제 (소유 검증). `deleteMany`/`updateMany` 로 일치 없으면 count=0, false 반환 (악의적 ID 시도 차단).

precision 결정:
- 명시값 (`forcePrecision`) 있으면 그것 사용
- 없으면 month 유무로 자동
- 명시 EXACT 인데 month null → APPROXIMATE 로 다운그레이드 (시각 일관성)

`/life-timeline/manage` — 전체 이벤트 목록 + 각 행에 수정/삭제. `/life-timeline/[eventId]` — 단일 수정 폼 (`getLifeEventById` 로 prefill).

`EventForm` — L2 폼과 다른 컴포넌트 (자유 카테고리 선택 + 앵커/사이 토글). 제출 후 `revalidatePath`.

검증 (`db/test-life-event-crud.ts`): 다른 사용자 삭제/수정 차단, ai_chat 행 절대 미수정, 정렬 (앵커 + 사이 섞임).

사이드 패널 메뉴 정리:
- "내 인생 연혁" (top, /life-timeline) — 새 메인
- "이번 달 타임머신" — currentMonthHref
- "내 기록" 제거 (연혁이 곧 내 기록)

`V3WelcomeBanner` — 첫 방문 시 한 번 "Lifebook 이 새로워졌어요. 이제 인생 연혁이 메인이에요" 안내. `localStorage("v3-welcome-seen")` 만, 새 DB 컬럼 0.

`/life-timeline` 상단 = 연혁 (주인공), 아래 보조 섹션 = "오늘의 한 걸음" 제목 + 가족 소식(0건 숨김) + 출석 + 진척. **시각적 위계 명확** (옅은 구분선 + 작은 섹션 제목).

## 6. L6 — AI 비서 모달

기존 v2 `AssistantPanel` **무수정** 재사용. `AssistantModal` 컴포넌트가 우측 상단 버튼 + 중앙 모달로 임베드:
- Esc/backdrop/X 닫기
- body.overflow = hidden (스크롤 잠금)
- 본문은 모달 안에서 스크롤 (`maxHeight: min(90vh, 100%)`)

맥락 결정 (page.tsx 에서 결정해 props 로):
- 가장 최근 life_event 의 `(eventYear, eventMonth ?? 6)` — 사용자가 막 본 연혁의 마지막 시기와 비서 컨텍스트 일치
- life_event 0 개 → LATEST (시드 마지막 달 2026.5) 폴백
- fallbackLabel: "2020년 5월 결혼" / "2026년 5월" — 사용자에게 *답이 어느 시기 기준* 인지 명시

"내 타임라인에 추가" (v2 의 keptEvents 의미) → 모달 닫고 `/life-timeline/add` 로 push (연혁 컨텍스트는 v2 의 (year,month) 의미와 안 맞아 L4 자유 추가 흐름으로).

저장된 답 prefetch — `listAssistantAnswers(userId, year, month)` 결과를 props 로 내려 모달 열자마자 "저장된 답변" 탭에 표시.

## 7. L7 — 첫 진입 흐름

**`/enter` (새 server component) — 로그인/동의 직후 도착하는 canonical 분기점.**

결정 규칙:
1. 인생 이벤트 ≥ 1 → `/life-timeline`
2. 인생 이벤트 0, 그 외 UserMemory ≥ 1 (v2 기존 사용자) → `/life-timeline` (빈 상태 EmptyState 가 부드러운 권유)
3. 둘 다 0 (완전 신규) → `/life-record?new=1` (랜딩 안내 배너)

판정: `lib/user-entry.ts:hasAnyUserMemory(userId)` — findFirst 1회로 createdVia 무관 어떤 행이라도 있으면 true. T6 + assistant 저장 + manual + Phase 7 ai_chat 모두 "v2 활동 흔적".

**왜 /life-timeline 자체에 게이트를 안 두나**
- 기존 사용자가 사이드 패널 "내 인생 연혁" 으로 돌아왔다가 인생 이벤트 0건이면 매번 /life-record 로 튕긴다. 길을 잃음.
- /enter 는 *처음 도착* 만 결정. 그 후 /life-timeline 은 빈 상태든 채워진 상태든 그대로.

진입 redirect 변경 (3곳):
- `app/login/page.tsx` — `redirectTo: "/timeline"` → `"/enter"`
- `app/consent/page.tsx` — 동의 완료 시 `redirect("/enter")`
- `app/consent/ConsentForm.tsx` — submit 후 `router.push("/enter")`

`/life-record` searchParams `{new}` — `?new=1` 일 때만 상단 violet 배너 한 줄. 닫기 ✕ = `/life-record` 링크 (param 없는 경로). 서버 searchParams 만으로 종료, **새 DB/localStorage/client 0**. V3WelcomeBanner 와 키·페이지·대상 사용자 다 달라 충돌 0.

## 10. 결정·트레이드오프 정리

- **새 모델 0 — UserMemory 한 곳에**: createdVia 디스크리미네이터로 통합. 가족 룸/반응/진척/공유 자동 호환. 다음 phase 에서 createdVia 별 view 가 필요해지면 그때 분리.
- **v2 코드 전체 보존**: T1~T6 (월 화면, 음악 카드, 시대 사건), V1~V4 (비서), 출석, 진척, 가족 반응 모두 그대로. *진입 동선만* 갈아끼움.
- **/enter 별도 페이지** vs /life-timeline 게이트: 별도 페이지가 사용자 흐름에 명확 (처음 도착 1회 분기, 이후 자유 이동).
- **랜딩 안내 = 서버 searchParams**: 새 DB/localStorage/client 0. 닫기 = param 없는 같은 URL 링크. 단순성 우선.
- **압박 금지 톤 일관**: "0개월=초대", "X 표시 X (zinc '아직')", "부정적 색·언어 0". 시니어 친화 원칙.
