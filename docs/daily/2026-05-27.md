## 1. 타임머신 v2 기획 — 정보 push → pull

기존 v1(T3~T6)은 그달 사건·음악을 전부 펼쳐 보여줬다. 실사용 통찰:
**읽을 게 너무 많아 흥미가 식는다.** 시스템이 "차려주는" 방식의 한계.

새 방향:
- 메인 화면 = **빈 기억 입력칸**(MonthStory 승격), 옆에 AI 비서.
- 정보는 시스템이 차리지 않고 사용자가 비서에게 묻는다(pull).
- 비서 답변 출처는 위험도로 분리:
  - **big 이벤트**(정치·사회·스포츠) → **검증 DB**(MonthEvent) 자신있게
  - **취향**(드라마·영화·유행) → **웹 검색** 조심스럽게 + 출처
  - **음악** → DB 차트 우선, 없으면 검색

기획 문서: `phase/타임머신_v2_AI비서_기획.md` (사용자가 직접 작성).

## 2. V1 — AI 비서 백엔드

### 라우팅 설계

키워드 기반 3분기 (AI 분류기는 비용·지연 이유로 보류):

| 카테고리 | 키워드 | 처리 |
|---|---|---|
| MUSIC | 노래·곡·가수·차트·OST·트로트 등 | `ChartSong` 그달 노출 → 있으면 템플릿(무료), 없으면 검색 |
| BIG | 사건·뉴스·정치·선거·올림픽·재난·IMF 등 | `MonthEvent` POLITICS_SOCIETY+SPORTS → 있으면 템플릿(무료), 없으면 검색 |
| TASTE | 드라마·영화·예능·만화·게임·유행·맛집 등 | 바로 검색 |

우선순위 MUSIC > BIG > TASTE. 매칭 실패 → TASTE로 폴백
(DB "없음"으로 오답하는 것보다 검색이 안전).

### Claude `web_search` 도구 연동

`lib/ai.ts`에 `chatWithWebSearch()` 추가. `chat()`는 손대지 않음 — 도구 사용은
별도 함수.

```ts
const tools = [{
  type: "web_search_20250305",
  name: "web_search",
  max_uses: opts.maxSearches ?? 1,
}];
```

응답 content는 `text` / `web_search_tool_result` 블록 혼합. text 본문 + 인용
(`citations` 또는 tool result `sources`) 모두 수집해 `{ text, citations[] }`
로 정리.

### 가드 시스템 프롬프트

```
- 가사·기사 문장 그대로 복제 금지. 사실만 우리 표현으로 요약.
- 단정 금지. "...였던 것 같아요" / "확인해보세요" 톤.
- 사용자 기억 대신 만들기 금지 (RAG 가드).
- 음악 답은 곡명·아티스트·순위만. 이미지·임베드 ❌.
- 시니어 친화 어조. 2~4문장 짧게.
```

- 검색 답 = `tokensFromUsage(in,out) + 1` (`chargeOneShot`에 `surcharge`
  파라미터 추가)

`chargeOneShot` 시그니처 surgical 확장 — 기존 voice_cleanup 호출부는 무영향.

```ts

```

## 3. MonthEvent 중복 정리

V1 (a) 검증 답에서 "한미 정상회담(워싱턴)" 이 두 번 출력 → 시드 중복 의심.

### 진단 (`db/diagnose-monthevent-dupes.ts`)

```ts
SELECT year, month, section, title, COUNT(*) FROM "MonthEvent"
GROUP BY ... HAVING COUNT(*) > 1
```

결과:
- **46개 중복 그룹**, 각각 정확히 2건씩
- 한쪽은 **deterministic id (24-hex sha256 슬라이스)**, 다른쪽은 **옛 cuid**
- 패턴: T2 시드 첫 실행은 cuid 기본값으로 들어갔고, H1 픽스(deterministic id) 후
  재실행으로 새 id 행이 추가됨. 같은 자연키 충돌 검사가 없었던 시점의 잔재.
- **사용자 추억 연결: 0건** — 자동 정리 안전.

### 정리 규칙

```
1. 그룹 내 추억이 여러 행으로 흩어져 있음 → skip + 경고
2. 추억이 1행에 있음 → 그 행 보존
3. 추억 없음 → deterministic 보존, 옛 cuid 삭제
4. 자동 판단 불가 → skip
```

이번엔 전부 케이스 3. 46행(옛 cuid) 삭제 → 중복 0 확인 + T6 통합 테스트
15체크 전부 통과.

## 4. V2 — 비서 UI

### 레이아웃

데스크톱 lg+ : 좌(기억칸=amber) / 우(비서=violet) 2단. 모바일: 세로.

```
┌──────────────────────────────┬─────────────────────────────┐
│ 2025년 8월, 당신의 이야기    │ 비서에게 물어보기            │
│ ┌────────────────────────┐   │ [이때 떠들썩한 일?] [노래?] │
│ │ (큰 textarea, 12행)    │   │ [드라마·영화?] [유행?] [물가?] │
│ └────────────────────────┘   │ ┌─[직접 묻기]───┐ [물어보기]│
│ [🎤 말로 적기] [✨ 다듬기]   │ 답변 ↓                       │
│ ── 내가 남긴 사건 ──         │ 본문 + SongCard + [추가] 버튼 │
│ • 한미 정상회담 [빼기]       │ 검증 자료라 무료 / 토큰 N개   │
└──────────────────────────────┴─────────────────────────────┘
            [이 달 저장하기]
   ← 2025년 7월             2025년 9월 →
```

### 재사용 vs 신규

| | |
|---|---|
| 재사용 (무변경) | `VoiceTextarea` `cleanupVoiceTextAction` `SongCard` `saveTimemachineMonthAction` `loadTimemachineMonth` 내비 가드 |
| 보존 (V2에서 미사용) | `MonthStory` `EventItem` `MonthForm` — 코드 유지, 화면에서만 빠짐 |
| 신규 | `AssistantPanel.tsx` `MonthV2.tsx` |
| 백엔드 확장 | `AssistantResult.events[]` / `songs[]` raw 추가 (UI 카드/버튼 렌더용) |

### 비서 → 타임라인 연결

1. BIG/DB 답마다 "내 타임라인에 추가" 버튼
2. 클릭 → `MonthV2.keptEvents` state에 `{ monthEventId, title, story: "" }` push
3. 좌측 amber 칩으로 표시 + "빼기" 가능
4. "이 달 저장하기" → `saveTimemachineMonthAction(year, month, { keptEvents, monthStory })`
   → T6 UserMemory 구조 그대로 (`createdVia = timemachine_event`)

검색 답(취향)은 카드/버튼 없이 본문 + 출처만 → 사용자가 직접 좌측 기억칸에 작성.

### 차감 안내

- DB 답: "검증된 자료라 토큰을 쓰지 않았어요."
- 검색 답: "토큰 N개 사용 · 남은 M개"
- 402 응답 → "토큰이 부족해요. [충전하러 가기]"
