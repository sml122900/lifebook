- `OnboardingForm.tsx`/`ProfileEditForm.tsx`: 정적 폼은 story kind 비표시(switch `case "story": return null` / `.filter`).

### F5 — 온보딩 장소 위젯 검색창 상시 노출 개편

기존 `PlaceableMultiItemWidget`은 항목 추가 후 "📍 지도에서 찾기" 버튼을 눌러야 검색창이 열렸다. 어르신에겐 한 단계가 더 있어 진입 장벽. 검색창을 처음부터 노출하도록 전면 재작성.

#### `app/onboarding-chat/widgets.tsx`
- `PlaceSearchInput` import 제거 — 위젯이 `/api/place-search` 엔드포인트를 직접 호출(같은 debounce 패턴 내장). `PlaceSearchInput.tsx` 무수정.
- 흐름: 검색창 **상시 노출, 네이버 기본** → "구글로 전환" 버튼 → 결과 클릭 시 좌표 포함 추가 / "📝 …검색 없이 추가" 또는 결과 없음 → 텍스트만 추가(좌표 null).
- 구글은 autocomplete 2-step(`action: "autocomplete"` → `action: "detail"`)로 좌표 획득.
- 추가 항목 칩: 좌표 있으면 emerald+📍, 텍스트만이면 amber.
- 어르신 친화: 검색창 text-[17px], 결과 버튼 min-h-[56px].
- 저장 정책 무변(좌표 있는 항목만 `residencePlaces`/`schoolPlaces` upsert).

### G2 — "그 시절 이야기"에 시대 데이터 직접 고르기(browse)

stories 칩 질문 shortcut 외에, 사용자가 시대 사건·음악 목록을 직접 보고 골라 담는 경로 추가.

#### 확인 결과(구현 전 조사)
- 조회 함수: `lib/era-events.ts`의 `listEraEvents`/`listEraSongs` 재사용(1980~2023 전체, 사건 88·음악 73). assistant의 `getMonthScreen`은 단월용이라 부적합.
- 생년 = `User.birthYear`(O). 지역 = `User.region`(존재하나 **시대 데이터에 지역 차원 없음 → 필터 불가**). birthYear만 사용.
- 시대 범위 UI = **연대 탭**(사용자 결정). 1980/90/2000/2010, birthYear+20로 기본 연대 자동 선택.

#### `app/timemachine/[year]/[month]/era-pick-actions.ts` (신규 server actions)
- `getEraCatalog()` — 전체 카탈로그 + birthYear 기반 `defaultDecade` 반환(데이터 작아 한 번에 내려주고 클라가 연대 필터).
- `addEraItemAsLifeEvent(kind, id)` — id로 서버에서 제목·연·월 재조회(클라 입력 불신뢰) → `createLifeEvent(isDraft=false)`. content는 비움(본인 회상 자리). 카테고리 `FAMILY` 기본(온보딩 story 추출·EventForm과 일관). `revalidatePath("/life-timeline")`.
- `/era`의 era_event(시대 배경, slate, 편집 불가)와 **별개 경로** — life_event(본인 사건, amber, 편집 가능)로 담김.

#### `AssistantPanel.tsx` browse 모드
- `era-selecting` 카드 재구성: 📋 목록에서 고르기(browse, 무료) / 🔍 AI에게 물어보기(ask, 기존).
- browse: 연대 탭 + 사건/음악 탭 + 항목별 "기억나요 +" 담기. lazy 카탈로그 로드.
- 담기 = 세션 옵티미스틱 "✓ 담음"(life_event는 monthEventId 미기록이라 영구 추적 X → 세션 한정).
- 빈 연대/탭 → "아직 준비 중" 안내(빈 화면 방지).
- `stories` 모드(우리 자료 채팅)는 **코드 보존** — 직접 진입만 빠지고 "저장된 답변" 경로에서 유지.

#### 검증
- `tsc --noEmit` 오류 0(세 작업 모두).
- 변경은 순수 additive — `era-events.ts`/`life-events.ts`/`PlaceSearchInput.tsx` 데이터·공용 레이어 무수정.
