# 2026-06-27 작업 일지 — 코치마크 온보딩 (메인 5단계 + 포스터 3화면, 공용 엔진)

어르신·가족 첫 사용자가 서비스를 바로 이해하도록 **오버레이 코치마크 온보딩**을
도입했다. 어두운 반투명 배경 + 핵심 버튼만 spotlight + 화살표 말풍선 + 단계 진행.
카카오·배달앱 첫 실행 가이드 형태. 기존 "사용법 물어보기" AI 챗(능동형 Q&A)과는
별개 — 이건 첫 진입 자동 코치마크다.

설계 핵심은 **공용 엔진 1개**(`CoachMarks`)를 만들고 화면마다 `steps` 배열 +
`data-tour` 속성만 더하는 것. 메인(`/life-timeline`)에 먼저 적용하고, 같은 엔진을
포스터 3화면(`/poster` → `/poster/select` → `/poster/view`)에 확장했다. 새 모델은
`User.completedTours String[]` 한 컬럼뿐(투어별 행 추가 X — id 만 늘림).

---

## 1. 공용 엔진 — `CoachMarks`

`app/components/CoachMarks.tsx` — `{steps, tourId, autoStart, onComplete}` 만 받는
재사용 컴포넌트. `lib/tours.ts` 에 순수 데이터(`CoachStep[]`)로 단계를 정의하고,
화면에는 `[data-tour="..."]` 속성을 단 요소를 둔다.

- **spotlight = 거대한 box-shadow 구멍** — 타겟 박스에 `box-shadow: 0 0 0 9999px
  rgba(0,0,0,.62)` 를 줘 *주변만* 어둡게. 타겟 자신의 `z-index` 와 무관해서 사이드
  패널(z-50) 위로도 구멍이 뚫린다(오버레이 z-60). 별도 클리핑·마스크 불필요.
- **단계별 사이드 패널 제어** — 콘텐츠 버튼 단계는 패널을 닫고, 패널 안 메뉴 단계는
  연다. `SIDE_PANEL_EVENT` 커스텀 이벤트로 `SidePanelLayout` 에 신호(투어 전용,
  localStorage 선호는 안 건드림). 모바일에서 패널이 본문을 가리는 문제 해결.
- **미존재 타겟 자동 skip** — 화면 상태에 따라 안 보이는 버튼은 `querySelector` 가
  null → 다음 단계로 건너뜀(상태별 분기 견고).
- 어르신 친화: 큰 글씨(text-lg/xl)·쉬운 말·큰 버튼(min-h-52)·한 번에 하나만.

## 2. 메인 코치마크 (5단계)

`/life-timeline` — ①AI 비서 ②한 장면 추가 ③이야기 나누기(핵심) ④내 토큰 ⑤포스터.
①②는 패널 닫힘, ③④는 패널 열림(묶어서 여닫이 최소화), ⑤는 다시 닫힘.

- 첫 진입 1회 자동(`completedTours` 에 `"main"` 없을 때 + 이벤트 있을 때). 이벤트
  0건이면 EmptyState 가 이미 안내라 투어 생략.
- 사이드 패널 "둘러보기 다시 보기" 로 재실행.

## 3. 포스터 코치마크 (3화면, 엔진 재사용)

포스터 흐름이 어르신에게 특히 어려워(노드/메모·편집) 화면마다 가이드를 붙였다.
**엔진·`data-tour` 패턴만 재사용** — `POSTER_*_STEPS` 정의 + 각 화면 속성.

| 화면 | tourId | 단계 |
|---|---|---|
| `/poster` | `poster-templates` | 디자인 고르기 / 맞춤형 |
| `/poster/select` | `poster-select` | **노드·메모**(첫 카드) / 고른 개수 / 시안 보기 |
| `/poster/view` | `poster-view` | 완성 시안 / 편집하기 / 주문하기 |

- 각 화면 첫 방문 1회 자동 + 좌측 하단 **"도움말" 버튼**(우측 AI FAB와 안 겹침)으로
  재실행. 공용 `ScreenTour`(도움말 버튼 + CoachMarks) 컴포넌트로 묶음.
- 재실행은 `START_TOUR_EVENT` dispatch — 한 화면에 CoachMarks 가 하나뿐이라 충돌 0.
- 포스터 단계는 패널과 무관(panel 필드 없음). 단 `begin()` 이 시작 시 패널을 닫아
  모바일에서 패널이 포스터를 가리는 것까지 자동 차단.

## 4. 데이터 모델

`User.completedTours String[] @default([])` — 마이그 `20260626140000`(ADD COLUMN,
기존 사용자 `'{}'`). 완료/건너뛰기 시 `markTourCompletedAction`(멱등·race-safe raw
SQL `array_append(array_remove(...))`)로 id 추가. **새 화면 투어는 컬럼 추가 없이
id 만 늘리면 됨**(reusable). `skippedLifeCategories` 와 같은 native array 패턴.

## 5. 검증 라운드에서 잡은 측정·위치 버그

코치마크의 진짜 난점은 "spotlight 가 타겟에 정확히 붙나"였다. 3차에 걸쳐 픽스.

### 메인 1·2차
- **rAF 폴링 측정** — 고정 지연 후 1회 측정의 오차(타겟보다 위 빈 공간)를 제거.
  타겟을 즉시 스크롤한 뒤 위치가 4프레임 연속 안정될 때까지(패널 단계는 최소
  520ms — 슬라이드 끝까지) 매 프레임 재측정. 모바일 토큰 카드가 패널 슬라이드 전
  off-screen 위치(`left:1049`)에 고정되던 조기 정지 버그 해소.
- **재실행 재설계** — RSC 재렌더·쿼리·key 의존을 버리고, CoachMarks 를 완료
  사용자에게도 항상 마운트(이벤트 수신용). 같은 페이지=`START_TOUR_EVENT`,
  다른 페이지=`?tour=main` 이동. `begin()` 이 완료 리셋 + 패널 닫기 + URL 정리.
- **ResizeObserver(body)** — 타임라인 등 비동기 렌더로 위 콘텐츠가 밀려 타겟이
  이동하는 늦은 layout shift 까지 재측정. spotlight 위치 transition 제거(측정값에
  정확히 스냅).
- **모바일 말풍선 고정** — 패널이 본문을 거의 덮는 모바일에서는 말풍선을 화면
  위/아래에 고정(패널 위 z-index)하고 화살표 생략. spotlight 링이 위치를 알림.

### 포스터 픽스
- **말풍선 위치 일반화(P1)** — 포스터 템플릿 카드는 세로로 길고 상단에 있어 "위에
  배치" 가 화면 밖으로 잘렸다(`top:-220`). 타겟 위/아래 **실제 여유 공간**(말풍선
  높이+여백)을 계산해 둘 다 부족하면 화면 위/아래 고정, 한쪽만 여유면 그쪽에.
  기존 `tall`(포스터 시안 A2 세로) 특수 케이스를 이 일반 로직이 흡수.
- **단계 전환 재스크롤(P2)** — 다음 타겟이 화면 밖이면 ring 이 사라지고 배경만
  남던 문제. tick 매 프레임 **타겟이 화면 밖이면 다시 `scrollIntoView`**
  (`behavior:"instant"` 강제 — 조상 `scroll-behavior:smooth` 영향 차단)해 재부착.
- **노드/메모 강조 범위(P3)** — `data-tour="poster-nodememo"` 를 버튼 행 → 사건
  카드 전체(`<li>`)로 이동해 제목·내용 포함 강조(맥락 자연스러움).

## 미완료 후속 / 짚어둘 점

- **정적(tsc·lint)까지만** 검증. 실제 동작은 화면으로 확인 필요:
  - 메인: 첫 진입 자동·5단계·재실행 시 패널 닫힘·모바일 ④⑤ ring.
  - 포스터: 3화면 자동/재실행·select 노드메모 카드 전체·view 세로 긴 시안 말풍선.
- 포스터 시안(`poster-preview`)은 A2 세로라 spotlight 구멍이 화면보다 커 딤이 거의
  안 보임 — 말풍선만 안내. 허용 범위(타겟이 곧 화면 전체).
- 엔진 재사용 검증 완료(메인→포스터). 추가 화면(예: `/people`·`/photos`)도 `steps`
  + `data-tour` 만으로 확장 가능.
- `ScreenTour` 도움말 버튼(좌하단)과 AI 비서 FAB(우하단) 공존 — 두 floating 버튼이
  많아지면 정리 후보.
- 첫 진입 자동은 **기존 사용자에게도 1회** 노출(completedTours 빈 컬럼) — 의도된
  소개. 텔레메트리(완료율·건너뛰기율·재실행율)는 후속.
