# Phase 7 — AI 대화로 추억 채우기 (가이드형 + 음성)

> **목표**: 확정한 트리거/앵커 옆에서 AI가 **가이드형 대화**로 추억을 끌어내 `UserMemory`로 저장한다. 시니어가 막막하지 않게, 텍스트와 음성 둘 다로 답할 수 있게.
> **선행 조건**: Phase 6 완료(확정/거부), Claude API 키, dev 서버 정상(단일 인스턴스).
> **분량**: 큼(두 번째 산). 핵심 루프(가이드 대화+텍스트)를 먼저 완성 후 음성을 얹는다.

**확정 설계**
- 대화: **가이드형** — AI가 구체적이고 따뜻한 질문 2~4개를 제시, 사용자가 골라 답(자유 입력도 허용).
- 입력: **텍스트 먼저 → 음성 나중**(별도 태스크로 분리).
- 톤: 인터뷰어/자서전 작가. 취조 아님. 열린 질문, 시니어 친화.

**불변 원칙 (CLAUDE.md)**
- **RAG 가드**: AI는 검증 DB(앵커/확정 트리거)에 있는 사건만 근거로 대화. **사건 창작 금지.**
- **UserMemory 첫 실제 쓰기 — 모든 read/write `userId` 스코프.** (Phase 3·5에서 약속한 지점)
- 민감정보 유도 금지, 솔로=비공개 기본.

**사용자가 미리 할 일**: `.env`에 `ANTHROPIC_API_KEY`. (.env.example에 빈 값 추가)

---

## 7.1 — Claude API 래퍼

**작업**
- `lib/ai.ts`: Anthropic Claude API 호출 래퍼(모델·max_tokens·에러 처리).
- 키는 `.env`에서, 코드 하드코딩 금지.

**완료 기준**: 테스트 프롬프트에 응답이 돌아온다.

---

## 7.2 — 추억 대화 진입 (트리거/앵커 카드 연결)

**목적**: Phase 6에서 "기억나요"로 확정한 카드(또는 앵커 카드)를 누르면 추억 대화가 열린다.

**작업**
- 확정된 TriggerResponse(또는 앵커 이벤트) 카드에 "추억 남기기" 진입점.
- `/memory/[eventId]` 또는 모달로 대화 UI 오픈.
- 해당 이벤트 정보(곡·아티스트·연도, 또는 사건명·연도)를 대화 컨텍스트로 전달.

**완료 기준**: 확정 카드에서 추억 대화가 열리고, 어떤 이벤트인지 대화가 인지한다.

---

## 7.3 — 가이드형 질문 생성 (RAG 가드 적용)

**목적**: AI가 그 사건/노래에 맞춘 따뜻한 질문 몇 개를 제시.

**작업**
- 프롬프트 설계: 입력 = 이벤트 정보 + 사용자 birthYear/당시 나이 (+ 가능하면 LifeProfile 일부).
- 출력 = **선택형 질문 2~4개** + "직접 쓰기" 옵션. 예: "광화문 연가가 나오던 그 시절, 어떤 게 떠오르세요?" / "그때 자주 가던 곳이 있었나요?" / "이 노래를 같이 듣던 사람이 있나요?"
- **RAG 가드**: 시스템 프롬프트에 "검증된 이 사건/노래만 다루고, 없는 사실을 지어내지 말 것. 사용자가 말한 내용만 추억으로 정리할 것" 명시.
- 질문은 열린형·시니어 친화 톤. 예/아니오 취조형 금지.

**완료 기준**: 이벤트에 맞는 선택형 질문이 생성되고, AI가 사건을 창작하지 않는다.

---

## 7.4 — 답변 → 추억 카드 저장 (UserMemory)

**목적**: 대화에서 나온 내용을 구조화해 저장.

**작업**
- 사용자가 질문에 답하면, AI가 답을 짧은 **추억 카드**로 정리(원문 보존 + 다듬은 요약).
- `UserMemory` 저장: `userId, eventId, year, month, title, content, createdVia="ai_chat"`.
- ⚠️ **반드시 `userId` 스코프**. 저장 후 타임라인 [내 사건] 레이어에 반영.
- AI는 사용자가 말한 것만 기록(없는 디테일 추가 금지).

**완료 기준**: 대화 후 UserMemory가 생성되고, 타임라인 내 사건 레이어에 카드가 나타난다.

---

## 7.5 — 대화 히스토리 저장

**작업**
- `AIConversation` / `AIMessage` 모델(이미 PRD에 있음) 마이그레이션 + `prisma generate` + 재시작.
- 이벤트별 대화 맥락 유지(이어서 답하기 가능).

**완료 기준**: 대화가 저장되고, 다시 들어가도 이어진다.

---

## 7.6 — 토큰 차감 훅 자리 마련

**목적**: Phase 8(결제) 대비. 지금은 차감 로직 없이 **훅 지점만** 만든다.

**작업**
- `lib/ai.ts` 호출 지점에 "여기서 토큰 차감"이라는 명확한 훅/주석 + 호출 횟수 로깅.
- 실제 차감·잔액은 Phase 8.

**완료 기준**: AI 호출 지점이 한 곳으로 모여 있고, Phase 8에서 차감을 끼울 자리가 분명하다.

---

## 7.7 — 음성 입력 (텍스트 루프 완성 후)

**목적**: 시니어 친화. **7.4까지 정상 동작 확인 후** 착수.

**작업**
- 브라우저 Web Speech API(또는 대체)로 음성 → 텍스트 변환, 답변 입력란에 연결.
- 큰 마이크 버튼, 명확한 녹음 상태 표시(시니어 접근성).
- 미지원 브라우저는 텍스트로 graceful fallback.

**완료 기준**: 음성으로 말한 내용이 텍스트로 입력되고, 안 되는 환경에선 텍스트로 무리 없이 대체된다.
