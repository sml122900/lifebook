# Phase 9.5 — 음악 재생 연결 (유튜브)

> **목표**: 음악 트리거 카드에서 그 곡을 **바로 들어볼 수 있게** 한다. 제목만 보는 것과 멜로디가 들리는 것은 회상 강도가 완전히 다르다 — 음악 트리거의 핵심 가치를 끌어올리는 기능.
> **선행 조건**: Phase 9(가족 공유) 완료. (독립적이라 순서는 유연)
> **분량**: 작음. (a) 검색 링크 방식이라 가볍다.
> **확장 계획**: 지금은 **(a) 유튜브 검색 링크**. 반응 좋으면 **(b) 임베드 플레이어**로 확장(별도 작업).

**왜 검색 링크부터인가**
- API 키 불필요(YouTube Data API 안 씀), 저작권/임베드 차단 부담 없음, 하루면 됨.
- 영상 ID를 미리 매칭·저장할 필요 없이 "곡명+아티스트"로 즉석 검색 → 가장 적합한 결과로 연결.

---

## 9.5.1 — "들어보기" 링크 생성 유틸

**작업**
- `lib/music/youtube.ts`에 함수: `youtubeSearchUrl(title, artist)` →
  `https://www.youtube.com/results?search_query=` + encodeURIComponent(`${title} ${artist}`).
- 검색어는 곡명 + 아티스트로 구성(연도는 넣지 않음 — 검색 정확도 저하 방지).
- ⚠️ 한글/특수문자가 URL에 안전하게 인코딩되는지(encodeURIComponent) 확인.

**완료 기준**: 임의의 곡·아티스트로 올바른 유튜브 검색 URL이 생성된다.

---

## 9.5.2 — 트리거 카드에 "들어보기" 버튼

**작업**
- `components/TriggerCard.tsx`에 **"▶ 들어보기"** 버튼 추가.
- 클릭 시 9.5.1 URL을 **새 탭**으로 열기(`target="_blank"`, `rel="noopener noreferrer"`).
- 버튼은 카드의 "기억나요/잘 모르겠어요" 흐름을 방해하지 않게 배치(별도 보조 버튼).
- 시니어 접근성: 큰 버튼, 명확한 ▶ 아이콘 + "들어보기" 텍스트 라벨 둘 다.

**완료 기준**: 트리거 카드의 "들어보기"를 누르면 새 탭에 그 곡의 유튜브 검색 결과가 뜬다.

---

## 9.5.3 — (선택) 추억/공유 화면에도 노출

**작업**
- 이미 저장된 음악 추억(UserMemory)이나 룸 공유 타임라인의 음악 카드에도 동일한 "들어보기" 버튼을 붙일지 결정.
- 음악 도메인(domain="music") 이벤트에만 노출되게 가드(앵커/세상 사건엔 안 뜨게).

**완료 기준**: 음악 관련 카드에서만 "들어보기"가 보이고, 그 외엔 안 보인다.

## 향후 확장 — (b) 임베드 플레이어 (지금 안 함)

반응이 좋으면 다음을 별도 작업으로:
- 곡별 유튜브 **영상 ID 매칭**(YouTube Data API 또는 큐레이션) → 카드 안 `<iframe>` 임베드 플레이어.
- 임베드 차단된 영상 fallback(차단 시 검색 링크로), 자동재생 정책·모바일 대응 고려.
- 영상 ID는 Event에 컬럼 추가(`youtubeVideoId`)해 캐시.
- ⚠️ 저작권/약관: 공식 임베드(iframe)는 유튜브 약관 내 허용 범위이나, 다운로드/재호스팅은 금지.
