## 1. 템플릿 피커 — 느티나무 + 인생 강물 (커밋 03513e5)

### 문제

T1 엔진은 한 종(느티나무)만 렌더했다. 종을 더하되 엔진을 안 건드려야 하고, river는 챕터가 고정 메타포명(상류~하구)이라 클러스터 라벨로 덮으면 안 됐다.

### 결정·구현

- `page.tsx`가 매니페스트 *리스트*를 map으로 각각 렌더 → `PosterInteractive`가 피커로 active만 표시(`key={active.id}` remount).
- river `idMap.chapter`를 **존재하지 않는 id(sentinel)**로 매핑 → `replaceTextById`/`hideById` no-op → 메타포명 보존. render.ts 무수정.
- river는 `significanceVariants`를 zelkova에서 재사용(슬롯·심볼 구조 동일). 계약은 grep으로 검증 — 폴더 위치·title/footer id 누락 2건이 실제 발견돼 태깅(비주얼 불변).
- `next.config.ts` `outputFileTracingIncludes`를 `templates/**/*.svg` 글롭으로 — river 런타임 fs 로드가 Vercel 번들에서 누락되던 배포 블로커 차단.

## 2. sephirot — 계약 불일치 STOP

디자이너 "100% 동일" 클레임을 grep으로 검증한 결과, 슬롯 DOM이 달랐다:

```
zelkova/river:                     sephirot:
<g id="slot-c1-e1" color>          <g id="slot-c1-e1" transform="translate(130,150)" color><use href="#node"/>
  <use href="#leaf-s" x="246"…/>     <g transform="translate(0,0)"><use href="#leaf-s" x="-3.5"…/></g>
</g>                                </g>
```

render.ts의 변형 주입 전제 3개가 모두 깨짐: ① 중첩 `<g>`(비탐욕 단일 `</g>` 매칭) ② 음수 로컬좌표(양수 앵커 정규식 미스) ③ group transform 포지셔닝(절대 `use x/y` 모델 불일치). naive 매니페스트로 강행 시 변형·S/M/L이 死하는 "조용히 망가진 템플릿" → STOP+보고. 자산만 커밋(e79a121), flat 재작업 후 합류.

## 3. C10 빈 슬롯 토큰 누수 (커밋 9291300)

river 빈 슬롯의 `{year}/{title}` 리터럴이 크롬에서 노출. 조사: server render는 빈 라벨에 `display="none"` 정상 부여(렌더 검증 보임 0), globals.css에 덮는 룰도 없음 → **근본 원인 미확인**(stale 배포 추정). 통합 effect 끝에 "{" 스윕(미주입 라벨을 인라인 `style.display:none`으로 차단)을 방어 가드로 추가. zelkova는 빈 라벨에 "{" 0개 → no-op(회귀 0). 차이의 본질: 둘 다 숨김이지만 zelkova 빈 내용 vs river `{토큰}`.

## 4. NUL 바이트 → git 바이너리 오인 (커밋 d98833e)

C10 fix 커밋이 `1 file changed, 0 insertions / Bin`으로 나왔다. 추적하니 메모 키 separator에 **U+0000(NUL)**이 섞여 git이 `.tsx`를 바이너리로 분류(diff/blame 깨짐). `perl -i -pe 's/\x00/ /g'`로 일반 공백 교체, 키 동작 동일. → 이후 커밋마다 `grep -caP '\x00'` 점검 루틴화. 상세: `docs/troubleshooting/git-binary-nul-byte.md`.

## 5. 편집 ① 위치 이동 (커밋 d6de71b)

"자세히 편집" 토글(기본 OFF — 어르신 auto 경로 보존). ON 시 채워진 사건 드래그.

- 포인터 이벤트(터치+마우스 통합 + `setPointerCapture`), `getScreenCTM().inverse()` + `createSVGPoint().matrixTransform`로 화면 px→viewBox 매핑(반응형 width:100% 스케일 자동 보정).
- `positions=Record<slotId,{dx,dy}>`. 슬롯+두 라벨에 인라인 `transform="translate"`(같이 이동). setAttribute라 CSS/presentation 충돌 0(C10 패턴).
- 라이브=`dragRef` 직접 갱신(리렌더 0), pointerup=state 커밋, 통합 effect 재적용(재주입 유지·idempotent). viewBox 클램프(getBBox).

## 6. 편집 ② 크기 조절 (커밋 e1fc587)

① transform 확장. `scales=Record<slotId,number>`(기존 S/M/L 변형 Map과 별도·공존).

- 합성 transform: `translate(dx,dy)·translate(cx,cy) scale(s) translate(-cx,-cy)` 중심 기준 균일 스케일. cx,cy=getBBox 중심.
- 핸들 대신 사건별 56px ±스테퍼(시니어 친화 — 1.2배 밀도 포스터의 작은 점 hit-test 취약). 클램프 0.5~2.0 끝에서 disabled.
- 위치 드래그 중 스케일 보존(dragRef 캡처) → 이동+리사이즈 동시.

## 7. 편집 ③ 포스터 메모 (미커밋→이번 문서 커밋과 함께)

가장 복잡(새 SVG 요소). 격리·가산으로.

- `posterMemos=[{id,text,x,y}]` — viewBox 자유 좌표(템플릿 무관·전환 유지).
- **별도 effect**가 `<g id="poster-memos">` 오버레이에 렌더(슬롯 effect와 격리, innerHTML 통째 교체 idempotent).
- 드래그는 `dragRef.kind:"slot"|"memo"` 분기 → ① 인프라 재사용, 슬롯 경로 무수정(①② 회귀 0). 텍스트 편집·삭제는 패널 리스트(시니어 친화), 위치는 SVG 드래그.
- "메모 모두 비우기"는 "위치·크기 초기화"와 분리(놀람 방지).
