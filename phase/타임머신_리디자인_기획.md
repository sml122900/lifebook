# Lifebook 핵심 경험 리디자인 — "타임머신" 방식

> **작성 배경**: 실사용 테스트 중 얻은 통찰. "기억을 채우는 과정이 재미있어야 끝까지 채우고, 그래야 책을 주문한다." 현재의 '대표 사건 하나' 방식은 이 재미를 못 살린다.
> **상태**: 기획 제안 (구현 전). 기존 Phase 5·6·7 타임라인/트리거/추억 흐름을 대체·보완하는 새 핵심 루프.

---

## 1. 무엇이 바뀌나 — 핵심 전환

| | 기존 (Phase 5~7) | 새 방식 ("타임머신") |
|---|---|---|
| 구조 | 출생연도 기준 타임라인을 펼쳐 보여줌 | **현재(2026.5)부터 한 달씩 거꾸로** 여행 |
| 사건 | 그 시대 대표 사건 위주 | **그달의 여러 사건을 나열** → 기억나는 것만 체크 |
| 진행 | 사용자가 알아서 탐색 | **"그달엔 이런 일이 있었어요, 당신은요?"** 로 이끎 |
| 입력 | 확정 카드에서 추억 대화 | **사건 옆에 바로 내 이야기** 작성(텍스트/음성) |
| 음악 | 세대 가중 RAG 추천 | **그달 차트 1~10위** (멜론/빌보드) |

**왜 더 나은가**
- **부담이 적다** — "인생 전체"가 아니라 "이번 달 하나"씩. 한 입 크기.
- **워밍업이 된다** — 가까운 기억(쉬움)부터 먼 기억(어려움)으로 자연스럽게.
- **능동적이다** — 나열된 것 중 고르고 지우니, 빈 화면 앞의 막막함이 없다.
- **중독성** — "다음 달엔 뭐가 있었지?" 한 달씩 넘기는 리듬이 계속 하게 만든다.

## 2. 핵심 화면 흐름

### 2.1 월(月) 단위 여행
- 시작: **2026년 5월** (현재 달). 위쪽에 "2026년 5월" 크게.
- 멘트: **"2026년 5월에는 이런 일이 있었어요. ○○님에겐 어떤 일이 있었나요?"**
- 한 달을 다 보면 **"← 이전 달(4월)로"** 버튼으로 거꾸로 이동.
- 진행률 느낌: "지금까지 N개월을 돌아봤어요" 같은 가벼운 표시(부담 주지 않게).

### 2.2 그달의 사건 — 나열 + 체크/삭제
- 그달의 big 이벤트들을 **목록으로** 보여줌(사회·문화·스포츠 등).
- 각 사건: **기억나면 체크(남김) / 안 나면 지움(X)**.
- 남긴 사건 옆에 **내 이야기 입력란** — "이 사건 때 나는…"
- 사건이 하나도 안 와닿으면 전부 지우고 **"이 달엔 특별한 게 없었어요"** 로 넘어가도 됨.

### 2.3 내 이야기 입력 (텍스트 + 음성)
- 텍스트 직접 입력.
- **음성 녹음 → AI가 다듬어 입력** — 말한 내용을 AI가 문장으로 정리(원문 보존 원칙 유지: 사용자가 말한 사실만, AI는 다듬기만).
- 시니어 친화: 큰 마이크 버튼, 명확한 녹음 상태.

### 2.4 그달의 음악 (차트 1~10위)
- 그달(집계 완료된 직전 달) **차트 1~10위**를 표시.
  - **국내: 멜론 월간 차트** / **해외: 빌보드 월간 차트**
- 곡: **순위 · 제목 · 아티스트** (텍스트만).
- 제목 클릭 → **유튜브 검색 결과로 이동하는 링크**(새 탭). 임베드 재생 X.
- 곡에도 "이 노래 기억나요 / 내 이야기" 연결 가능.

## 3. 음악 — 차트 기반으로 단순화

**기존 RAG(세대 가중 추천)에서 → 차트 1~10위로 변경**
- 기준: 그달의 **멜론 월간 1~10위(국내)**, **빌보드 월간 1~10위(해외)**.
- 노출: 한 달에 **5~10곡**만 (과하지 않게).
- 집계 시점 주의: 예) 지금이 5월이면 **4월 월간 차트는 집계 완료** → 4월 1~10위 사용. (현재 진행 중인 달은 차트 미확정이라 직전 달 기준.)

**장점**
- 정확하고 객관적("그달 1등") — AI가 추천을 지어낼 여지 0.
- RAG 임베딩·세대 가중 로직이 불필요해져 **단순·견고**해짐.
- 차트는 사실(순위)이라 분쟁 소지 적음.

**과거로 갈수록**
- 멜론은 2004년 서비스 시작 — 그 이전 시대는 다른 출처 필요(예: 가요톱텐, 연간 가요 차트 등). **과거 차트 데이터 확보는 별도 과제**(시드 작업).

## 3.5 음악 카드 디자인 — 이미지 없이 감성 살리기

**원칙: 남의 이미지를 가져오지 않고, 우리가 디자인으로 만든다.**
- 가수 사진 ❌ (사진 저작권 + 초상권/퍼블리시티권 — 앨범 커버보다 권리가 한 겹 더 많아 더 위험)
- 앨범 커버 ❌ (응용미술저작물 별도 저작권, 서버 저장 시 복제권 침해)
- → 둘 다 안 쓴다. 대신 아래 요소로 충분히, 오히려 더 우리답게.

**디자인 요소 (저작권 0)**
- **순위 타이포** — 순위 숫자를 디자인 주인공으로. 1위는 크게/강조색, 아래로 갈수록 작게.
- **순위 원형 배지** — 1위는 강조색 원, 2~10위는 차분한 색. 음악 앱 느낌.
- **시대의 색** — 그달이 속한 시대의 무드를 카드 색감으로:
  - 1980년대: 빛바랜 세피아 / 1990년대: 청록·핫핑크 네온 / 2000년대: Y2K 메탈릭 / 2010s~: 미니멀 톤 등
  - "그 시절"을 색으로 환기 — 가수 사진보다 더 회고적이고, 경쟁사엔 없는 차별점.
- **매체 모티브** — LP판·카세트테이프·라디오를 직접 그린 SVG 또는 오픈 라이선스 아이콘.
- **재생 아이콘(▶)** — 클릭 시 유튜브 검색으로 이동.
- **(선택) 순위 변동** — "전월 대비 ▲3" 같은 차트 느낌.

**검증됨**: 텍스트 나열(삭막) < 타이포 중심 < 시대의 색(추천). 이미지 없이도 음악 앱 수준의 감성 가능.

## 4. ⚠️ 저작권 — 반드시 지킬 선

**음악 (링크)**
- ✅ **유튜브 검색 결과로 이동하는 링크**만 사용(클릭 시 유튜브로 나감). 안전.
- ❌ **임베드 재생(우리 화면 안에서 바로 재생) 금지** — 한국 법원이 임베디드 링크를 침해 방조로 본 판례 있음. 정 필요하면 공식 YouTube IFrame API + 법무 검토.

**앨범/곡 커버 이미지 — MVP에서는 쓰지 않는다 (권장)**
- 앨범 커버는 **응용미술저작물로 별도 저작권**이 형성될 수 있음. 우리 서버에 저장(복제)하면 **복제권 침해** 소지.
- 비영리·출처표시를 해도 면책 안 됨(허락이 원칙).
- **결론: 텍스트(순위·제목·아티스트)만으로 충분.** 커버 이미지는 리스크 0인 길을 택한다. 향후 정식 라이선스(멜론/스포티파이 API 등) 확보 시 재검토.

**차트 데이터 (순위)**
- "4월 1위 = ○○○" 같은 **순위 사실 자체는 저작권 대상 아님**.
- 단, 특정 사이트의 **차트 표·디자인을 그대로 복제**하면 편집저작물/DB권 이슈 가능 → **순위·곡명·아티스트라는 사실만** 우리 형식으로 재구성해 저장.

**시대 사건 (앵커)**
- 사건 사실은 저작권 없음. 단, 특정 기사·백과의 **서술 문장 복제 금지** → 사실만 우리 표현으로.

## 5. 기존 설계와의 관계

**대체/변경**
- Phase 5(출생연도 타임라인) → **월 단위 역방향 여행**으로 재구성.
- Phase 6(음악 RAG 추천) → **차트 1~10위**로 단순화. 임베딩·세대가중 로직은 보류/제거 후보.

**유지**
- Phase 7의 **추억 저장(UserMemory) + 음성 입력 + RAG 가드(AI는 사용자 말만 다듬음)** 는 그대로 유효. 입력 위치만 "사건 옆"으로 옮김.
- Phase 9(가족 공유)·결제·동의 구조는 영향 없음.

**데이터 과제 (새로 필요)**
- 월별 사건 데이터: "2026년 5월엔 무슨 일이 있었나"를 월 단위로 채운 DB. (기존 앵커 32개보다 훨씬 촘촘해야 함)
- 월별 차트 데이터: 멜론/빌보드 월간 1~10위를 월별로. 과거로 갈수록 출처 확보 난이도↑.
- 이 두 가지가 **새 방식의 핵심 자산이자 진입장벽(해자).**

## 6. 단계적 구현 제안 (작게 검증)

1. **최근 12개월만 먼저** — 2025.6~2026.5의 월별 사건 + 차트로 프로토타입. 데이터 확보가 쉬운 최근부터.
2. 월 넘기기 UX + 사건 체크/삭제 + 내 이야기 입력(텍스트) 검증.
3. 음성 입력 → AI 다듬기 추가.
4. 음악 차트 1~10위 + 유튜브 검색 링크 연결.
5. 반응 좋으면 **과거로 확장**(연도별 데이터 시드를 거슬러 채움).

> 핵심 검증 질문: "사용자가 한 달 한 달 넘기며 계속 채우고 싶어 하는가?" — 이게 되면 책 주문으로 이어진다.

## 6.5 음악 데이터 수집 — 방법과 근거

**데이터 단위: 월간으로 확정 ✅**
- 이유: **"이 노래 나왔을 때 진짜 난리였지"** — 노래가 *터진 그 시점*이 기억의 트리거다. 연간 평균으로는 그 폭발의 순간이 뭉개진다. 월간이라야 회상이 살아난다.
- 연간/분기 대안은 데이터 수집은 쉽지만 회상 효과가 약해 기각.

**수집 방법: 멜론 월간 차트를 보고 수작업 시드 입력 ✅**
- 양: 월 10곡 × 12개월 = 120곡(국내). 한 사람이 1~2시간이면 정리 가능. 자동화 이득이 거의 없는 수준.
- 방법: 멜론 월간 차트를 *눈으로 보고* "순위·곡명·가수"만 우리 시드 형식(`seed-music-triggers.ts`)으로 직접 입력.
- **표기 통일 검수** — 유튜브 검색 링크가 "곡명+가수"로 만들어지므로, 표기가 정확해야 원곡이 제대로 뜬다. 입력 시 한 번 검수.

**검토했으나 채택 안 한 방법 (근거 기록 — 같은 고민 반복 방지)**
| 방법 | 왜 안 쓰나 |
|---|---|
| 멜론 자동 스크래핑 | 이용약관의 "자동화 접근 금지" 위반. 차트 표 복제 시 편집저작물/DB권 이슈. |
| 크롬 AI 스크린샷 | 사람 열람과 법적 성격 다름 — "자동화 수단"으로 분류돼 약관 위반 가능. 반복 시 명백한 자동수집. 양도 적어 수작업이 더 빠름. |
| Spotify Web API | **차트 API 폐기됨**(2024.11 deprecated, CSV는 2024 초 중단). 곡 메타데이터는 제공하나 *차트 순위*는 못 받음. |
| 나무위키 | **상업적 이용 금지(CC BY-NC-SA)** — 결제 서비스에 복사 불가. 월간도 없음(연간 TOP10만). 참고만 가능. |
| 위키피디아 | 상업 이용 가능(CC BY-SA, 출처 표기). 단 월간 TOP10은 약함(주간 1위 위주). **해외(빌보드) 보조 자료로 유용.** |

**출처별 라이선스 요약**
- 멜론 = 순위 *사실*만 보고 우리 표현으로 재입력 (스크래핑·표 복제 금지)
- 나무위키 = 복사 금지(상업 NC). 흐름 파악용 참고만.
- 위키피디아 = 출처 표기 시 활용 가능. 해외 차트에 유용.

**범위 전략**
- **MVP: 국내(멜론 월간)부터.** 핵심 타깃이 한국 어르신 → 한국 노래가 가장 강한 트리거.
- 해외(빌보드)는 이후 확장. 위키피디아 연표를 보조로.
- 과거(2004년 이전, 멜론 시작 전)는 장기 과제 — 가요톱텐 등 다른 출처 필요.

## 7. 열린 질문 (정해야 할 것)

**결정됨**
- ~~데이터 단위 월간 vs 연간~~ → **월간 확정** (회상 효과)
- ~~음악 수집 방법~~ → **멜론 월간 보고 수작업 시드** (위 6.5)

**미결**
- 월별 *사건*은 어디서? (뉴스 아카이브, 위키 연표 등 — 사실만 재구성. 음악과 별개 과제)
- 과거 차트(2004년 이전 국내)는 어떤 출처로? (가요톱텐 등 — 장기 과제)
- 사건 목록의 분량(한 달에 몇 개가 적당한가) — 너무 많으면 압도, 적으면 빈약.
- 해외 음악을 MVP에 넣을지, 국내만으로 먼저 검증할지.
