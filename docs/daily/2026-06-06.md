# 2026-06-06 — 월 화면 비활성화 (메인 동선에서 '월' 개념 제거)

짧은 한 세션. 큰 구조 결정 하나.

**통찰**: "작년 이벤트 순서는 기억해도 몇 월이었는진 기억 안 난다." 매달 회고 빈 칸을 채우라는 부담은 v3 인생 연혁의 핵심 가치(상대 좌표·연 단위)와 정면 충돌. v3 피벗 때 메인은 연혁으로 옮겼지만 *클릭 동선* 은 아직 월 화면으로 갔다. 이번 세션에서 그 잔재를 끊었다.

코드는 **지우지 않고 진입로만 차단** — 사용자 명시. 시대 사건/음악 시드는 비서가 여전히 참조하므로 절대 손대지 않음.

---

## 결정한 비활성화 방식 (계획 단계)

사용자가 옵션 중에서 골랐다.

| 항목 | 선택 | 사유 |
|---|---|---|
| `/timemachine/[year]/[month]` 라우트 | **`/life-timeline` 으로 redirect** | 직접 URL·옛 북마크·외부 링크 모두 안전하게 흡수. 가장 명확. |
| 사이드 패널 "이번 달 타임머신" 메뉴 | **그냥 제거** | "월" 단어 안 보이게. 메뉴 5개로 축소. |
| 연혁 이벤트 클릭 동선 | **`/life-timeline/[eventId]/edit`** | 이미 존재. 이야기·장소·인물 통합 편집. |
| 코드 보존 | **archived 함수 + 시드 무수정** | 부활 시 default export 만 교체. |

## 변경 5곳

### 1. `app/timemachine/[year]/[month]/page.tsx` — redirect + archived 보존
```ts
export default function TimemachineMonthPage() {
  redirect("/life-timeline");
}

// 이하 _TimemachineMonthPageArchived 함수, prevMonth/nextMonth/FilledBadge,
// LATEST_YEAR/MONTH/EARLIEST 상수 모두 그대로. import 도 그대로.
async function _TimemachineMonthPageArchived(...) { ... }

// eslint-disable-next-line @typescript-eslint/no-unused-vars
const __preserve_archived_exports = {
  _TimemachineMonthPageArchived, prevMonth, nextMonth, ...
};
```

`__preserve_archived_exports` 트릭 — archived 함수와 그 안에서만 쓰는 헬퍼·상수 전부를 객체로 한 번 참조해 `no-unused-vars` ESLint 경고 회피. 한 줄 disable 로 정리. 부활 시: default export 만 archived 로 가리키면 끝.

### 2. `app/timemachine/SidePanel.tsx` — MenuItem 한 줄 삭제 + 타입에서 currentMonthHref 제거
- "이번 달 타임머신" `<MenuItem>` 한 블록 삭제
- `SidePanelData` 타입에서 `currentMonthHref: string` 필드 제거
- 메뉴 6 → 5개 (인생 연혁 / 인물록 / 가족 룸 / 회원정보 / 설정)

### 3. `lib/side-panel-data.ts` — 헬퍼에서 currentMonthHref 빌드 제거
- `SidePanelDataPrepared` 타입에서 필드 제거
- `LATEST_YEAR/MONTH` 상수도 같이 제거 (사용처 0)

### 4. `app/life-timeline/TimelineView.tsx` — 점·카드 Link href 변경
- `timemachineHref(e)` → `editHref(e)` (이름·내용 모두)
- 새 함수: `editHref(e: RenderEvent) → /life-timeline/${e.originalId}/edit`
- isPeriodEnd 행도 `originalId` 라 "끝" 점 눌러도 같은 이야기를 연다 (H4 originalId 명시 픽스 덕)
- aria-label "그 시기의 타임머신 열기" → "이 이야기 편집하기"
- `APPROX_DEFAULT_MONTH` 상수 미사용 → 제거 (page.tsx 의 비서 fallback 용은 그대로)

### 5. `app/timemachine/ProgressCard.tsx` — Link → div (시각만, 클릭 X)
- 12개월 그리드 각 칸이 `/timemachine/[year]/[month]` 로 가던 진입로 차단
- `<Link>` → `<li>` (border/색은 그대로 유지 — 채움 amber, 빔 회색)
- 안내문 "달별 기록 (눌러서 그 달로 이동)" → "달별 기록" (클릭 안 됨을 자연스럽게)
- aria-label: 인터랙티브 → 정보성으로 (`"기록 있음, 보러 가기"` → `"기록 있음"`)

## 회고

이번 결정의 무게는 코드 양이 아니라 *모순 해소* 에 있다. v3 피벗 때 메인은 옮겼지만 "월" 진입로가 곳곳에 남아 사용자가 결국 "정확한 월" 을 떠올려야 하는 경험으로 다시 끌려갔다. 이번엔:

- **사이드** ("이번 달 타임머신" 메뉴)
- **연혁** (점·카드 클릭 → 월 화면)
- **진척** (12개월 그리드 칸 → 월 화면)
- **라우트 자체** (직접 URL)

네 진입로 모두를 한 번에 닫았다. 진입로가 있는 한 사용자는 그쪽으로 흘러간다 — 부분 차단은 부분 노출이다. 동시에 "삭제 아님, 보존" 원칙으로 archived 함수 + 시드 데이터를 다 살려, 나중에 AI 채팅에서 시대 사건/음악 DB 를 활용하거나 월 화면을 부활시킬 때 한 줄 교체로 복구 가능하게 했다.

`__preserve_archived_exports` 객체 패턴은 처음 써 봤는데, "사용 0 함수를 ESLint 경고 없이 한 파일에 남기는" 깔끔한 트릭. 차후 다른 deprecated 경로에 재사용 가능.

## 후속 (이연)

- `/timemachine/[year]/[month]` 라우트의 실제 URL 진입 텔레메트리 — 외부 링크/북마크가 얼마나 도달하는지 (redirect 흡수율 확인 후 라우트 자체 제거 시점 결정)
- `AssistantPanel` 의 `(year, month)` 컨텍스트 — life_event 기반은 OK 지만 "특정 시기 외 전반적 질문" 모드도 추가 검토 (CLAUDE.md L7 후속과 결합)
- ProgressCard 자체의 의미 — v3 인생 연혁이 메인이라 "월별 진척" 의 가치가 약화. "연도별 이야기 수" 같은 v3 친화 카드로 재구성 후보
- 사이드 패널 메뉴 5개로 줄었으니 시각적 여백 정리 (현재는 단순 제거만)
