## 1. 헤더 "타임머신" 버튼 제거

### 배경

v3 월 OFF(2026-06-06) 때 사이드 패널 "이번 달 타임머신" 메뉴·진척 그리드 진입로는 닫았지만, **root layout 헤더의 "타임머신" 버튼**이 잔재로 남아 있었다. 누르면 `/timemachine` → `/life-timeline` redirect 라 기능상 무해하지만, 어르신에겐 "타임머신을 눌렀는데 연혁이 나온다" = 혼란.

### 작업

- `app/layout.tsx` — 타임머신 `Link` 블록(7줄) 삭제 + 헤더 주석 갱신. 변경 파일 1개
- `/timemachine` 라우트·redirect·archived 코드 전부 무수정 (빌드 결과에 라우트 살아있음 확인)
- 가족 룸·토큰·계정 메뉴·로그인 버튼 무영향

이로써 메인 동선의 타임머신 입구 = **0개**.

## 2. 첫 방문 환영 카드 (최소 온보딩)

### 배경

신규 사용자는 `/enter → /life-record?new=1` 로 가지만, 거기서 건너뛰고 `/life-timeline` 에 오면 EmptyState 외 인사·안내가 없다. 첫 행동 하나만 제시하는 1회성 카드 필요.

### 핵심 결정 — `onboardingCompletedAt` 재사용 (마이그 0)

| 후보 | 판단 |
|---|---|
| 새 컬럼 (`welcomeDismissedAt` 등) | 마이그 필요 — 기존 필드로 충분해 기각 |
| localStorage (V3WelcomeBanner 패턴) | 브라우저별 — "사용자당 1회" 보장 안 됨 |
| **기존 `User.onboardingCompletedAt`** | ✅ 채택 |

재사용 안전 근거:
- 설정하는 곳 = 레거시 `/onboarding` (Phase 4) 한 곳, 체크하는 곳 = 레거시 `/timeline` 한 곳
- 신규 v3 사용자는 `/enter → /life-record` 동선이라 이 필드가 **영원히 null** — "첫 방문" 신호로 정확
- 환영 카드 닫기로 찍으면 레거시 `/timeline` 의 온보딩 리다이렉트도 안 타게 됨 — 부작용이 아니라 바람직
- 이미 찍힌 사용자(레거시 온보딩 완료)는 `updateMany where onboardingCompletedAt: null` 로 원래 시각 보존

### 구현

- **`app/life-timeline/WelcomeCard.tsx`** (신규, client) — 👋 + "{이름}님, 환영해요. 인생의 한 장면부터 하나 적어볼까요?" 시니어 친화: 큰 글씨(3xl~4xl), [시작하기] 64px amber / [닫기] 56px 중립색, 압박색 X
  - [시작하기] → dismiss 액션 후 `/life-timeline/add` push
  - [닫기] → 옵티미스틱 숨김 + dismiss 액션
- **`app/life-timeline/welcome-actions.ts`** (신규, server action) — `dismissWelcomeAction`: auth + `updateMany {id, onboardingCompletedAt: null} → set now()` + `revalidatePath("/life-timeline")`
- **`app/life-timeline/page.tsx`** — `Promise.all` 에 user select 1개 추가(fetch 5개 병렬). 표시 조건 `showWelcome = onboardingCompletedAt == null && !hasEvents` (연혁 있는 기존 사용자에겐 안 뜸)

### V3WelcomeBanner 동시 노출 방지

신규 사용자는 localStorage 도 비어 있어 V3 배너("Lifebook 이 새로워졌어요")가 같이 뜸 — 배너 두 장 연속은 시니어 부담 + 신규에겐 "새로워졌어요" 무의미. 해결 두 겹:
- 렌더: `{showWelcome ? <WelcomeCard /> : <V3WelcomeBanner />}` — 동시 노출 0
- 닫을 때: WelcomeCard 가 `v3-welcome-seen` localStorage 도 함께 찍음 — "환영 카드 닫자마자 다음 배너 등장" 시퀀스 차단

### 검증

- **`db/test-welcome-card.ts`** (신규, 4 시나리오) — 신규 사용자 조건 true → 닫기 1행 갱신 → 재방문 조건 false → **중복 닫기 0건 + 원래 시각 보존**. 임시 사용자 생성/삭제로 본 데이터 무영향
- `next build`(tsc 포함) + `test-life-events` 회귀 통과

## 3. 트러블슈팅 — dev 서버 중 `next build` → `.next` 충돌

환영 카드 검증으로 `next build` 를 돌렸는데 사용자 dev 서버가 떠 있었다. prod 빌드가 같은 `.next` 폴더를 덮어써 dev 서버가 stale 화 — 브라우저에 "Next.js 16.2.6 **(stale)** / Jest worker encountered 2 child process exceptions" 런타임 에러.

복구: `Get-NetTCPConnection -LocalPort 3000` 으로 점유 PID 확인 → `Stop-Process` → `.next` 삭제(1차 삭제는 자식 프로세스 잠금으로 잔존, 2초 후 재시도 성공) → dev 재시작 (Ready 823ms, 로그인→/enter→/life-timeline 정상 흐름 확인).

상세: `docs/troubleshooting/dev-server-build-next-conflict.md`. 재발 방지 작업 수칙: **dev 떠 있는 동안 `next build` 금지** — 빌드 검증 필요 시 dev 먼저 정지.
