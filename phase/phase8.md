# Phase 8 — 토큰 결제 (MVP 닫기)

> **목표**: AI 사용량 비례로 토큰을 차감하고, 토스페이먼츠 **테스트 모드**로 토큰 구매(충전) 흐름을 완성한다. 이걸로 핵심 루프에 결제가 붙어 MVP가 닫힌다.
> **선행 조건**: Phase 7 완료(AI 호출 + `[ai] in/out/total` 로깅 존재).
> **작업 방식**: 8.1부터 하나씩, 완료 기준 충족 시 커밋 후 다음.

**확정 설계**
- 차감: **세밀** — 실제 AI input/output 토큰 비례. (7.6 로깅을 기준점으로)
- 결제: **토스페이먼츠 테스트 모드** 실연동(실제 청구 없음).

**사용자가 미리 할 일**: 토스페이먼츠 **테스트** 클라이언트키/시크릿키를 `.env`에 (`TOSS_CLIENT_KEY`, `TOSS_SECRET_KEY`). .env.example엔 빈 값.

**⚠️ 보안·안전 원칙 (꼭 지킬 것)**
- 결제 **금액·승인은 반드시 서버에서 검증**한다. 클라이언트가 보낸 금액을 그대로 신뢰 금지(위변조 방지).
- 시크릿키는 **서버 전용**, 절대 클라이언트 번들/URL에 노출 금지.
- 카드 정보는 토스 결제창이 처리 — 우리 서버/DB에 카드번호를 저장하지 않는다.
- 모든 지갑/거래 read/write는 `userId` 스코프.

---

## 8.1 — 토큰 환산 정책 정의

**목적**: "AI 토큰 사용량 → 우리 서비스 토큰"의 환산 규칙을 한 곳에 둔다.

**작업**
- `lib/tokens/policy.ts`에 환산 상수 정의. 예: `서비스토큰 = ceil((in + out) / N)` (N은 조정 가능 상수).
- 가입 시 **무료 지급량**, 추억 1건 평균 소모 기준치도 상수로.
- 7.6 로깅에서 실제 평균 in/out을 보고 N과 무료량을 현실적으로 잡는다.

**완료 기준**: AI 사용량 → 차감 토큰 계산이 한 함수로 정해진다(상수만 바꾸면 정책 조정 가능).

---

## 8.2 — 지갑 모델 + 무료 지급

**작업**
- `TokenWallet`(`userId @unique, balance`) + `TokenTransaction`(`userId, delta, reason, refId, createdAt`) 모델 + 마이그레이션 + `prisma generate` + 재시작.
- 신규 가입 시 무료 토큰 지급(트랜잭션 기록).
- ⚠️ 잔액은 **항상 트랜잭션 합으로 검증 가능**하게(감사 추적).

**완료 기준**: 가입하면 지갑이 생기고 무료 토큰이 트랜잭션과 함께 지급된다.

---

## 8.3 — AI 호출에 실제 차감 연결

**목적**: 7.6에서 만든 훅 자리에 실제 차감을 끼운다.

**작업**
- AI 호출 성공 후 `in/out` 토큰 → 8.1 정책으로 환산 → 지갑 차감 + 트랜잭션 기록.
- 차감은 **호출 응답이 성공한 뒤**(실패한 호출엔 차감 안 함).
- 재진입 시 AI 호출 0(7.5 캐시)이면 차감도 0인지 확인.

**완료 기준**: 추억 대화 후 사용량만큼 잔액이 줄고 트랜잭션이 남는다.

---

## 8.4 — 잔액 부족 처리

**작업**
- AI 호출 전 잔액 확인. 부족하면 호출 막고 **충전 유도 UI**.
- 시니어 친화: 큰 글씨로 "토큰이 부족해요. 충전하시겠어요?" + 큰 버튼.

**완료 기준**: 잔액 0/부족 시 AI 호출이 막히고 충전 안내가 뜬다.

---

## 8.5 — 토스페이먼츠 테스트 결제 (충전)

**목적**: 토큰 구매 흐름을 테스트 모드로 실제처럼.

**작업**
- 충전 상품(토큰 패키지) 정의(예: 1,000원=N토큰). 금액·토큰량은 서버 소스에 고정.
- 토스페이먼츠 결제위젯/SDK로 결제창 호출(클라이언트키는 클라이언트, 시크릿키는 서버).
- 결제 성공 콜백 → **서버에서 결제 승인 API 호출로 금액 검증** → 검증 성공 시에만 토큰 적립(트랜잭션 기록).
- 테스트 카드로 끝까지 흐름 확인.

**완료 기준**: 테스트 결제 완료 → 서버 검증 통과 → 토큰이 잔액에 적립된다.

---

## 8.6 — 결제/충전 내역 화면

**작업**
- 지갑 잔액 + 거래 내역(지급/차감/충전) 표시.
- 시니어 친화: 한눈에 보이는 잔액, 큰 글씨.

**완료 기준**: 사용자가 잔액과 내역을 확인할 수 있다.
