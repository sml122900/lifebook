# 2026-05-23

## 한 줄 요약

Phase 0~3 진행. Next.js 16 + Postgres/pgvector + Prisma 7 + Auth.js v5 풀스택 베이스를 세우고, 32개 앵커 이벤트 시드 → 연도순 타임라인 렌더 → Google OAuth + 명시 동의 게이트 + 라우트 보호까지 한 사이클 완성.

## Phase별 진행

### Phase 0 — 프로젝트 셋업

- `create-next-app`으로 Next.js 16 (TS / Tailwind / ESLint / App Router / Turbopack) 초기화
- `pgvector/pgvector:pg17` Docker 컨테이너 + `vector` 확장
- Prisma 7 + `@prisma/adapter-pg` driver adapter, `lib/db.ts` 싱글톤
- `.env` / `.env.example` 분리, `.gitignore`에 `!.env.example` 예외 추가
- Prettier + `format` / `format:check` 스크립트

### Phase 1 — 데이터 모델 + 앵커 시드

- `User`, `LifeProfile`, `Event`, `UserMemory` 4모델 + `EventTier` / `EventCategory` enum
- 1979 → 2025 검증된 앵커 이벤트 **32개** 시드, 멱등 재실행 가능 (`delete + createMany`)
- pgvector embedding 컬럼은 Phase 6로 연기(주석으로 의도 명시)

### Phase 2 — 타임라인 정적 렌더

- `/timeline` 서버 컴포넌트가 Prisma로 직접 조회
- 연도별 그룹핑 + 세로 타임라인 + 2레이어(세상 사건 / 내 사건) 레이아웃
- 7개 domain 컬러 뱃지(`EventCard`), 시니어 접근성 패스(본문 18px+, 카드 88px, 4px focus ring)
- 빈 상태("이 시절, 당신은 어떤 일이 있었나요?")

### Phase 3 — 인증 + 동의 게이트

- Auth.js v5 (`next-auth@beta`) + Google OAuth + `@auth/prisma-adapter`, 세션은 JWT
- `Account` / `Session` / `VerificationToken` 모델 추가, `User`에 동의 타임스탬프 3종
- `/consent` — 3개 필수 + 1개 선택 체크박스, 사전 체크 없음, 서버 액션이 `*ConsentAt` 기록
- `proxy.ts` (Next 16의 middleware 후속) 라우트 보호: 미로그인→`/login`, 미동의→`/consent`
- 전역 헤더 + 로그아웃 버튼, `/login` Google 시작 버튼
