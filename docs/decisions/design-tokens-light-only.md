# 결정 — 디자인 토큰 시스템 (라이트 온리)

2026-06-12. 디자이너방 토큰 가이드 v1.0 적용 시리즈(1~5차).
기존 `dark-mode-token-swap.md` 의 다크모드 결정을 **대체(supersede)** 한다.

## Problem

- 화면 전반의 색이 zinc/amber/white 하드코드로 분산 — 페이지마다 미묘하게
  다른 회색·흰색, 일관 정책 부재.
- 다크모드(CSS 변수 swap)가 모든 신규 화면에 "다크 대응" 부채를 지움 —
  실제로 v3 신규 12+ 파일 다크 미대응(M2)이 후속 목록에 계속 쌓였다.
- 핵심 타깃(시니어)에게 다크모드 수요 사실상 0. 회상 서비스의 감성은
  "따뜻한 종이 톤" 단일 무드가 더 적합.
- 버튼·칩·빈 상태가 페이지마다 제각각 구현 — 위계(주요/보조/파괴적)와
  터치 크기(48px+)가 코드 리뷰로만 지켜지는 상태.

## Action

### 결정 1 — 다크모드 폐기, 라이트 단일 팔레트

`ThemeToggle.tsx`·`theme-actions.ts`(쿠키) 삭제, `:root { color-scheme:
light }` 고정. OS 다크 설정과 무관하게 항상 라이트. 토큰이 단일 값이라
"50↔950 대칭" 같은 swap 설계 자체가 사라짐 — 신규 화면의 다크 대응
부채 원천 제거.

### 결정 2 — 토큰 네이밍: 크기 키워드 금지 (base → canvas)

가이드 명칭 "base" 를 `--color-base` 로 정의하면 Tailwind v4 가
`text-base`(글자 크기 유틸리티)를 **색상 유틸리티로 덮어써** 글자색=
배경색 사고. 색 토큰명에 base/lg/sm 등 크기 키워드 금지 → `canvas`.

### 결정 3 — 버튼 위계 5종 + 필 색 단일 원칙 (`components/ui/Button`)

- `primary` = bg-action 필 — **화면당 1개** (주요 CTA).
- `secondary`(brand 보더+action 글자) / `tertiary`(line 보더+ink-soft) /
  `plain`(텍스트형) / `destructive`(**tertiary 형태에 danger 글자만 —
  빨강 필 금지**, 실제 탈퇴 확인 단계 같은 최종 관문에서만 danger 필).
- size md=48px / lg=56px — 시니어 터치 최소를 컴포넌트가 보장.

### 결정 4 — 선택 칩은 필 금지

선택 = `bg-banner + border-brand + text-action`, 비선택 = `bg-surface +
border-line + text-ink-soft`. action 필은 버튼(행동) 전용 — 칩(상태)에
필을 쓰면 "눌러야 할 것"과 "선택된 것"이 구분 안 됨. 진행바 fill 은 brand.

### 결정 5 — 연대 틴트는 워시 전용

decade 틴트 9종 + 스트립 9종. **텍스트 배경 금지** — 스트립(4px)·점·
5~10% 워시만, 틴트 위 텍스트는 ink/ink-soft 만. 저채도 틴트 위 본문
텍스트 대비 미달을 규칙으로 차단.

### 의도적 비치환

음악 rose / 정보 sky / SPORTS blue 등 **의미 분리용** 보조 팔레트와
사이드 패널 amber 강조는 잔존 — 의미색까지 토큰으로 흡수할지는 후속.

## Result

- 100여 파일 치환, 5차 grep 검증 `bg-white|text-black|ThemeToggle` 잔존 0.
- 본문 Pretendard + 제목 Noto Serif KR 로 서체 위계 통일.
- tsc·기존 검증 스크립트 회귀 0.

### 트레이드오프

- 다크모드 복원 시 토큰 값 이원화부터 다시 설계해야 함 — 수요 확인 전엔
  지불하지 않기로 한 비용.
- 보조 팔레트(rose/sky 등) 이원 체계 잔존 — 토큰 단일 진실 원천은 아직
  표면색·위계까지만.

### 일반화된 학습

1. **타깃이 좁으면 모드(다크/라이트)는 부채다** — 모든 신규 화면에
   세금을 매기는 기능은 수요 증거가 생긴 후에.
2. **유틸리티 생성형 CSS 에서 토큰명은 네임스페이스 충돌 검토 필수** —
   색 토큰이 크기 유틸리티를 덮어쓸 수 있다.
3. **위계·터치 크기는 컴포넌트가 보장** — 가이드 문서가 아니라 Button
   variant/size 가 규칙의 집행자.
