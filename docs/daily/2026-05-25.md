## 1. 회원 탈퇴 (PIPA 동의 철회권)

[basket 2 후보](./../par-materials.md)에서 미뤘던 항목을 본격 구현.

### 결정사항 (사전 질의)

| 항목 | 결정 |
|---|---|
| Owner 룸 처리 | **최고참 consent 멤버에 자동 이전**, 없으면 cascade 삭제 |
| 공동 SharedMemory | createdBy SetNull → "탈퇴한 사용자" 표기 |
| 결제 기록 (paid TokenOrder) | userId SetNull 익명화 후 5년 보존 (전자상거래법) |

### 스키마 변경 (`20260524064511_add_withdrawal_support`)

- `LifeProfile`·`UserMemory.user` → `onDelete: Cascade` 추가 (기존 default Restrict → delete 시 막힘)
- `SharedMemory.createdById` → **nullable + SetNull**
- `TokenOrder.userId` → **nullable + SetNull** (paid 보존 패턴)

### 액션 트랜잭션 (`app/account/delete/actions.ts`)

```ts
await prisma.$transaction(async (tx) => {
  // 1) 소유 룸 — 최고참 consent 멤버에게 이전, 없으면 룸 삭제
  for (const room of ownedRooms) {
    const successor = room.members[0];
    if (successor) {
      await tx.sharedRoom.update({ where: { id: room.id }, data: { ownerId: successor.userId } });
      await tx.roomMember.update({ where: { id: successor.id }, data: { role: "owner" } });
    } else {
      await tx.sharedRoom.delete({ where: { id: room.id } });
    }
  }
  // 2) 미결제 주문만 삭제 (paid 는 cascade SetNull 로 익명화)
  await tx.tokenOrder.deleteMany({ where: { userId, status: { not: "paid" } } });
  // 3) UserMemory 향 가족 룸 댓글 사전 정리 (polymorphic FK 라 자동 cascade 안 됨)
  await tx.comment.deleteMany({ where: { targetType: "user_memory", targetId: { in: memoryIds } }});
  // 4) user.delete → 나머지 cascade/SetNull
  await tx.user.delete({ where: { id: userId } });
});
```

### 다크모드 — CSS 변수 swap (zero component churn)

`globals.css` 한 파일만 수정해 전체 적용:

```css
@custom-variant dark (&:where(.dark, .dark *));

.dark {
  color-scheme: dark;
  /* 중립 (zinc/white/black) 토큰 뒤집기 — bg-white·text-zinc-900 등이 자동 반전 */
  --color-white: oklch(0.205 0 0);
  --color-zinc-900: oklch(0.97 0 0);
  /* ... zinc-50..900 모두 ... */

  /* 의미색 50↔950 대칭 swap — bg-rose-50 위 text-zinc-900 가독성 회복 */
  --color-rose-50: oklch(0.271 0.105 12.094);  /* 깊은 로즈 */
  --color-rose-900: oklch(0.941 0.030 12.580); /* 옅은 로즈 */
  /* ... amber/emerald/sky/violet/blue 같은 패턴 ... */
}
```

핵심:
- Tailwind v4 의 `var(--color-*)` 기반 유틸 → 변수만 redefine 하면 자동 반영
- `text-zinc-900` 가 다크에서 밝아지는데 의미색 배경(`bg-rose-50`)이 안 뒤집히면 라이트 위 라이트 → 안 보임 → **대칭 swap** 으로 해결
- 토글: 쿠키 (`lifebook-theme`) + 서버에서 `<html class="dark">` 주입 → SSR 무깜빡임. JS 없이 form action 으로 동작

### Turbopack CSS HMR 부재 트러블슈팅

globals.css 변경이 자동 반영 안 되어 `.next` 캐시 삭제 + 완전 재시작 필요. 별도 트러블슈팅 문서로
정리 → `docs/troubleshooting/turbopack-css-stale.md`.

### 입력창 강제 다크 처리

`color-scheme: dark` 만으로는 Windows Chrome 의 일부 `<input>`/`<textarea>` 가 흰 배경 유지.
`globals.css` 에 `.dark input[type="..."] / textarea / select { background-color: ...; color: ...; }`
일괄 규칙 추가 — 컴포넌트 파일 수정 0건.

## 5. 자체 코드 검토 + 후속 픽스

타임머신 T1~T6 코드만 6 관점 (버그·회귀·결제·저작권·일관성·시니어 UX) 검토. 16건 발견 → 즉시 6건 픽스, 나머지 10건은 통합 테스트 후로 미룸.

### 픽스한 6건

| 항목 | 변경 | 영향 |
|---|---|---|
| **H1** | 시드 deterministic id (SHA-256 `section\|year\|month\|title`) per-row upsert | 시드 재실행해도 `UserMemory.monthEventId` 보존 |
| **H2** | `cleanupVoiceText` 빈/동일 응답 시 `inputTokens=outputTokens=0` → `chargeOneShot` 차감 스킵 | 변화 없으면 토큰 안 잃음 (Anthropic 비용은 운영 흡수) |
| **H3** | `EARLIEST_YEAR/MONTH = 2025/6` 추가, 이전 달 버튼 가드 | 시드 밖 무한 과거 이동 차단 |
| **M2** | EventItem 취소 시 `onStoryChange("")` 제거 | 메모 보존, 다시 "남기기" 누르면 복원 |
| **L2** | billing `REASON_LABEL` 에 `voice_cleanup` | 거래내역에 한글 표시 |
| **L4** | page.tsx stale 주석 (T6 UserMemory N+1 행) | 최신화 |

## 회고

- **피벗을 한 세션에 완성**. T1 → T2 → T3 → T3 보완 → T4 → T5 → T6 → 검토 → 픽스가 사용자 phase 진행 규칙 ("하나 끝나면 멈추고 알려") 의 흐름 위에서 매끄럽게 굴러갔음. 도중 사용자가 발견한 UX 결함 (저장 안 됨, 마이크 없음, 진입 버튼 없음) 도 같은 사이클 안에서 흡수.
- **다크모드는 CSS 한 파일 + 의미색 swap** 으로 컴포넌트 파일 0건 수정. 시간 절약 + 회귀 위험 0. zinc 만 뒤집고 끝나면 안 됨 — text+bg 가 같이 뒤집혀야 함을 사용자 화면 캡처 보고 발견.
- **자체 검토 후 픽스 분리** 가 좋았음. 진단 16건 → 사용자가 시급 6건 골라 줌 → 그 6건만 정확히 픽스. "지금은 진단만, 코드는 안 고침" 룰을 처음부터 명시한 게 효과적.
