# Phase 2 — 타임라인 정적 렌더 (첫 보이는 화면)

> **목표**: 로그인·온보딩 없이, Phase 1의 앵커 이벤트를 **연도순 세로 타임라인**으로 화면에 띄운다.
> 제품의 첫 "와" 순간. 시니어도 편하게 읽히는 것이 1순위.
> **선행 조건**: Phase 1 완료 (Event 테이블에 앵커 이벤트 적재됨).
> **작업 방식**: 2.1부터 하나씩, 완료 기준 충족 시 커밋 후 다음.

**이 단계의 범위 / 비범위**
- ✅ 한다: 앵커 이벤트를 연도순으로 렌더, 2레이어 레이아웃, 시니어 접근성, 빈 상태.
- ❌ 안 한다: 인증(Phase 3), 온보딩(Phase 4), 개인화 필터(Phase 5), 트리거 이벤트(Phase 6).
  → 사용자 출생연도는 **임시 상수로 하드코딩**한다.

---

## 2.1 — /timeline 라우트 + 데이터 조회

**목적**: 서버에서 앵커 이벤트를 연도순으로 읽어와 페이지에 넘긴다.

**작업**
- `app/timeline/page.tsx`를 **서버 컴포넌트**로 작성, Prisma로 직접 조회:
  ```ts
  const events = await prisma.event.findMany({
    where: { category: "anchor" },
    orderBy: [{ year: "asc" }, { month: "asc" }],
  });
  ```
- 임시 상수: `const DEMO_BIRTH_YEAR = 1990;` (개인화는 Phase 5에서)

**완료 기준**: `/timeline` 접속 시 서버에서 이벤트가 조회되어 페이지로 전달된다(콘솔/렌더로 확인).

---

## 2.2 — 연도순 세로 타임라인 렌더

**목적**: 이벤트를 연도 기준으로 묶어 위→아래로 흐르는 연표를 만든다.

**작업**
- 이벤트를 **연도별로 그룹핑**(같은 해 여러 이벤트는 월순 정렬).
- 각 연도 구간에 큰 연도 헤더(예: `1997`)를 두고, 그 아래 해당 연도 이벤트 카드들을 배치.
- 세로 중심선(타임라인 축) + 연도 마커로 "시간이 흐른다"는 시각적 흐름을 준다.

**완료 기준**: 1979 → 2025 순으로 연도가 흐르고, 각 연도 밑에 사건이 보인다.

---

## 2.3 — 2레이어 레이아웃 (세상 사건 / 내 사건)

**목적**: 타임라인이 두 층으로 읽히게 한다 — 한쪽은 세상 사건, 한쪽은 내 추억.

**작업**
- 데스크톱: 중심 축을 기준으로 한쪽(예: 좌)에 **세상 사건**(앵커), 반대쪽(우)에 **내 사건**(UserMemory) 트랙.
- 모바일: 위아래로 자연스럽게 쌓이되 두 레이어가 색/라벨로 구분되게.
- 지금은 **내 사건 트랙은 비어 있음**(데이터 없음) → 2.6의 빈 상태로 채운다.

**완료 기준**: 화면에서 "세상 사건"과 "내 사건" 두 영역이 시각적으로 명확히 구분된다.

---

## 2.4 — 이벤트 카드 컴포넌트

**목적**: 사건 하나를 보여주는 재사용 컴포넌트.

**작업**
- `components/EventCard.tsx`: 표시 항목 = 연·월, 제목, (있으면) 설명, domain 뱃지.
- `domain`별 색상 뱃지 매핑(`kr_politics` / `kr_society` / `disaster` / `sports` / `tech` / `economy` / `world`).
- 카드는 클릭 가능한 형태로 만들되, 클릭 동작(추억 추가)은 Phase 7에서 연결 → 지금은 hover/focus 스타일만.

**완료 기준**: 모든 앵커 이벤트가 카드로 렌더되고, domain 뱃지 색이 구분된다.

---

## 2.5 — 시니어 접근성 스타일 (최우선)

**목적**: 고령 사용자가 편하게 읽고 누를 수 있게 한다. 이 단계의 핵심 품질 기준.

**작업**
- 본문 글자 18px 이상, 연도 헤더는 크고 굵게.
- 고대비 색 조합(연한 회색 텍스트 금지), 충분한 줄간격·여백.
- 클릭/터치 영역 최소 44×44px.
- 모바일 단일 컬럼에서도 흐름이 무너지지 않게.

**완료 기준**: 작은 화면·큰 글씨에서도 타임라인이 깨지지 않고 읽기 쉽다.

---

## 2.6 — 빈 상태 UI (내 사건)

**목적**: 내 사건이 아직 없을 때, 다음 행동을 부드럽게 안내한다.

**작업**
- 내 사건 트랙(또는 각 연도의 내 사건 칸)에 빈 상태 메시지:
  예) "이 시절, 당신은 어떤 일이 있었나요? 곧 추억을 더할 수 있어요."
- 강요·죄책감 유발 문구 금지. 따뜻하고 가벼운 톤.

**완료 기준**: 내 사건이 없을 때 빈 상태 안내가 자연스럽게 보인다.

---

## 2.7 — (선택) 흐름 보조

**목적**: 긴 연표의 탐색을 돕는다. 시간 남으면.

**작업(원할 경우)**: 우측 연도 점프 네비, 스크롤 시 현재 연도 표시(sticky 헤더) 등.

**완료 기준**: 선택 사항이므로 건너뛰어도 무방.
