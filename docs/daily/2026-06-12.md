## 1. 디자인 토큰 시스템 (시리즈 1~5차 완결)

### 배경

부모님 테스트를 앞두고 화면 전반의 색이 zinc/amber/white 하드코드로 흩어져 있어 일관성·시니어 가독(따뜻한 종이 톤·고대비) 모두 약했다. 디자이너방 토큰 가이드 v1.0 을 받아 시리즈로 적용.

### 토큰 정의 (1차, `app/globals.css @theme`)

- 표면: `canvas`(페이지 배경 #faf7f0) / `surface`(카드 #fffdf8) / `banner`(강조 박스 #fff8e8)
- 텍스트: `ink`(#2a2520) / `ink-soft` / `ink-faint` · 보더: `line`(#e5decf)
- 의미색: `brand`(골드 #c8923d) / `action`(버튼 필 #8a5f1a) / `action-hover` / `sage` / `danger` / `success`+`success-deep`
- 연대(decade) 틴트 9종 + 스트립 9종 — **텍스트 배경 금지**, 스트립(4px)·점·5~10% 워시만. 틴트 위 텍스트는 ink/ink-soft 만
- 서체: 본문 Pretendard Variable(npm `pretendard`), 제목 h1/h2 = Noto Serif KR(`next/font` 변수 → `@theme inline` 참조)

**토큰명 함정**: 가이드 명칭 "base" 를 그대로 쓰면 `--color-base` 가 Tailwind 글자 크기 `text-base` 를 **색상 유틸리티로 덮어써** 글자색=배경색 사고. `canvas` 로 개명 (메모리 노트 보존).

### 다크모드 폐기 (결정)

기존 "CSS 변수 swap 다크모드" 결정을 뒤집음 — 모든 신규 화면이 다크 대응 부채(M2 후속 누적)를 지는 구조였고, 핵심 타깃(시니어)에게 다크모드 수요가 사실상 0. `ThemeToggle.tsx`·`theme-actions.ts` 삭제, `:root { color-scheme: light }` 고정. 상세: `docs/decisions/design-tokens-light-only.md`.

### 공용 컴포넌트 (2차)

- `components/ui/Button.tsx` — 위계 5종: `primary`(bg-action 필, **화면당 1개**) / `secondary`(brand 보더+action 글자) / `tertiary`(line 보더+ink-soft) / `plain`(텍스트형) / `destructive`(tertiary 형태에 danger 글자만 — **빨강 필 금지**). size md=48px/lg=56px (시니어 터치 최소). `Button`/`ButtonLink`/`buttonClasses` 3 형태
- `components/ui/EmptyState.tsx` — lucide 아이콘 + 메시지 + action 필 버튼. people/photos 빈 상태에 적용 (페이지별 중복 구현 제거)

### 전면 치환 (3~5차)

- 100여 파일에서 `bg-white`/`text-zinc-*`/`border-zinc-*` → 토큰. 5차 검증 grep: `bg-white|text-black|ThemeToggle` 잔존 0
- root layout 개편 — 구 헤더(잔액·UserMenu·theme) 제거, **사이드 패널을 root 글로벌로 승격**(`loadSidePanelData` + `SidePanelLayout`, 비로그인 미렌더)
- 연대 배경(`lib/era-labels.ts`) — raw amber/emerald 워시 → decade 틴트 10% 워시 (틴트 규칙 준수)
- 설정 페이지 2건 — "토큰 화면 열기" 버튼 primary 통일(필 색 단일 원칙), 회원 탈퇴 섹션 de-danger(평시 중립 카드 + destructive 플레인, 실제 탈퇴 확인 단계의 danger 필은 유지)
- **선택 칩 스펙** — 선택: `bg-banner + border-brand + text-action`(**필 금지**) / 비선택: `bg-surface + border-line + text-ink-soft`. 진행바 fill 은 brand. ProfileEditForm·OnboardingForm·EraView·AssistantPanel 적용
- 의도적 잔존: 음악 rose / 정보 sky / SPORTS blue 등 의미 분리용 보조 팔레트, 사이드 패널 amber 강조 — 토큰화 여부는 후속 결정

## 2. 문장 다듬기 — MVP(맞춤법) → Lv2(문장 정리)

### 원칙 (전 단계 공통)

- **원문 영구 보존** — 교정본은 `refinedText` 별도 컬럼. `displayRefined=true`(사용자가 [이대로 바꾸기]) 일 때만 표시 스왑. "원래 글 보기" 토글 상시
- **무료** — Haiku 호출, `chargeOneShot` 없음 (Anthropic 비용 운영 흡수)
- content 수정 시 refined 3필드 초기화 — life-events 는 실제 content 변경 시에만(장소·제목만 고치면 교정본 보존), era-stash 는 무조건(content 항상 교체)

### 4단계 구현

1. **스키마** — `UserMemory.refinedText String? / refinedAt DateTime? / displayRefined Boolean @default(false)` (마이그 `20260612000000_memory_refine_spelling`, ADD COLUMN 3줄 기존 행 무영향)
2. **API** — `lib/memory-refine.ts`(`refineMemorySpelling`/`applyRefined`/`discardRefined`, 소유 가드 = updateMany {id,userId}) + `POST /api/memory/[memoryId]/refine`(auth 401 → 검증 400 → 404 → 친화 한국어 에러). NO_CHANGE sentinel + 공백 정규화 동일성 비교로 무변경 시 저장 0
3. **UI** — `RefineSection.tsx`(edit 화면 EventForm children, 회상 아래·사진 위). [글 다듬기] secondary+Pencil → 전/후 카드 세로([원래 글]/[다듬은 글] 라벨) → [이대로 바꾸기] primary / [그대로 두기] tertiary. no_change 토스트 "고칠 곳이 없어요. 잘 쓰셨어요!"(success 토큰). 결정 안 한 교정본은 재방문 시 review 복원. 18px+·48px+ 시니어 규격
4. **표시** — `getLifeEvents` 가 `displayRefined && refinedText` 면 content 자리에 교정본 반환 → 연혁·비서 컨텍스트 등 호출자 0줄 자동 적용. `getLifeEventById` 는 **원문 유지** + refined 필드 별도 노출(편집 textarea 는 항상 원문 — 교정본 재편집 사고 차단)

### Lv2 — 문장 정리 통합

프롬프트 확장: 허용 = 맞춤법·띄어쓰기·구두점 + **군말(어/음/그니까/인제 등) 제거 + 긴 문장 분리 + 반복 정리 + 비문 교정**. 절대 금지 = 내용 추가·요약·생략, 사투리·입버릇·고유명사 변경, 1인칭 변경. 서버측 길이 가드 = 결과가 원문의 **0.6~1.2 배** 벗어나면 왜곡 의심 → no_change(저장 안 함).

### 트러블슈팅 — 길이 가드 vs 군말 제거 충돌 + 사투리 표준화

첫 스펙(0.8 하한 + 프롬프트 "80~120%" 수치)으로 실측하니 군말 많은 표본(123자)이 67% 로 줄어 **하한에 차단** — 군말 비중이 큰 글일수록(=가장 다듬기가 필요한 글) no_change 가 나오는 모순. 또 모델이 "어머이"→"어머니" 로 사투리를 표준화.

→ 하한 0.6 완화 + 프롬프트 수치 삭제("요약하지 마라. 군말 제거 외에는 문장이나 내용을 빼지 마라") + 사투리 예시 조항(어머이·어무이·억수로·~했당께·~카더라·그라모, "군말과 사투리는 다르다") 추가. 재실측: 76% 통과, "어머이"·"억수로"·"타가지고" 보존 + 군말 전부 제거. 상세: `docs/troubleshooting/refine-length-guard-filler.md`.
