# Lifebook v3 — "인생 연혁" 방식 (매달 부담 → 인생 골격)

> **작성 배경**: v2(타임머신/AI 비서) 실사용 통찰. *"월마다 말을 남기는 게 너무 부담스럽다."* 빈 칸을 매달 채우라는 건 일기 강박과 같아, 어르신껜 더 무거움. 사람은 인생을 "1998년 7월"이 아니라 *"대학 다닐 때"* 식으로 떠올린다 — 절대 좌표가 아니라 상대 좌표로.
> **새 방향**: **인생 연혁(가로 시간축)**이 메인. 정해진 질문으로 인생의 *골격*을 잡고, 그 위에 큰 사건들을 사용자가 *원할 때만* 붙임. "매달 채우기"가 아니라 "내 인생의 큰 줄기 한 번 잡기".
> **상태**: v2 코드는 *대체하지 않고 보조로 살림*. 월별 타임머신은 깊이 파고 싶을 때 들어가는 보조 도구로 재배치.

---

## 1. 핵심 전환

| | v2 (월별 타임머신) | v3 (인생 연혁) |
|---|---|---|
| 메인 화면 | 월별 빈 칸을 채워 가는 방식 | **가로 시간축의 인생 연혁** |
| 입구 | 매달 빈 칸 부담 | "인생 기록하기" → 정해진 질문으로 골격 잡기 |
| 시간 표현 | (year, month) 절대 좌표 필수 | **앵커(정확) + 사이(상대 좌표)** 혼합 |
| 부담 | 높음 (매달 채워야) | 낮음 (큰 줄기만, 사이는 대략) |
| 월별 화면 | 메인 | **보조** (특정 시기 깊이 파고 싶을 때) |

**왜 더 나은가**
- 사용자가 *정확한 월*을 안 떠올려도 됨. "대학 입학과 군 입대 사이" 정도면 충분.
- 인생 전체가 *한눈에* 보임 → 성취감과 진척이 직관적.
- 매달 빈 칸의 *압박*이 사라짐. 큰 사건 중심으로 *원할 때만* 추가.

- **가로 시간축** — 첨부 레퍼런스 스타일. 시니어 친화 큰 점·큰 글씨.
- **앵커 이벤트(정확)** — 진한 큰 점. 정확한 연/월(또는 연도)이 있는 사건.
- **사이 이벤트(대략)** — 작은 점/점선. 추정 연도만, "정확하지 않음" 시각 구분.
- **AI 비서 버튼** — 우측 상단. 평소엔 닫혀있다 누르면 기존 채팅(v2 비서) 띄움.

## 3. 인생 골격 잡기 — 정해진 질문 흐름

**로그인 후 첫 진입** 또는 **"인생 기록하기" 버튼** 누르면 시작.

**질문은 정해진 순서로 차례차례** (사용자 답하기 → 다음 질문 → ... 빠지면 건너뛰기 가능).

### 질문 카테고리 (예시 — 시드 후 사용자 검토)

1. **출생** — 어느 지역에서 태어나셨어요? / 출생 연도는?
2. **유년기** — 어디서 자라셨어요? (이사가 있었다면)
3. **학교** — 초/중/고를 어디서 다니셨어요? 입학·졸업 연도?
4. **20대 활동** — 대학·취업·군대(남성) 등
5. **첫 직장** — 어디서 일 시작하셨어요? 언제?
6. **연애·결혼** — 결혼하셨다면 언제, 누구와? (선택)
7. **자녀** — 자녀가 있다면 몇 분, 언제 태어났는지? (선택)
8. **직업 변화** — 이직·창업·은퇴 등 큰 변화
9. **거주 변화** — 큰 이사
10. **기타 큰 사건** — 큰 병/사고, 종교, 취미 등 (선택)

각 질문은 **건너뛰기 가능**. 어르신이 답하기 싫거나 모르면 패스. 강요 없음.

### 질문 흐름의 원칙
- 한 번에 한 질문씩, 큰 글씨로.
- 답하기 어려우면 "잘 기억 안 나요 / 답하기 싫어요" 버튼.
- 중간 저장 — 다음에 와서 이어서.
- 다 마치면 골격 완성 → 가로 연혁표로 시각화.

## 4. 시간 표현 — 앵커와 사이

### 앵커 이벤트 (정확한 시점)
- "1972년 3월 초등학교 입학", "2020년 5월 결혼"
- 사용자가 *연/월을 정확히 답한* 사건
- 진한 큰 점으로 시간축의 정확한 위치에 표시

### 사이 이벤트 (상대 위치)
- "대학교 1학년 때 친구들과 여행"
- 사용자가 *정확한 시점은 모르지만* 앵커들 사이에 있다는 건 아는 사건
- **저장 방식**: 추정 연도 + `precision: "approximate"` 플래그
- AI 대화: "OO 추억은 어느 시기였어요?" → "대학 1학년쯤" → (이전 답에서 대학 입학=2020 알고 있으니) "그럼 2020년쯤이겠네요?" → `year=2020, precision=approximate` 저장
- 시각적으로 작은 점/약한 색으로 *앵커 사이*에 표시. 어르신이 "이건 정확, 이건 대략"을 한눈에 구분.

### 데이터 모델 — UserMemory에 얹기
- 새 모델 만들지 말고 기존 UserMemory에 `createdVia: "life_event"` 디스크리미네이터로.
- 필드 추가 (additive only):
  - `eventTitle` (예: "대학교 입학")
  - `eventYear` (Int, 추정이라도 채움)
  - `eventMonth` (Int?, 모르면 null)
  - `precision` (enum: EXACT / APPROXIMATE)
  - `category` (출생/학교/직업/가족 등)
- 기존 UserMemory를 쓰는 가족 룸·스탬프·진척 시각화가 *자동으로* 작동.

## 5. 이야기 첨부 — 연혁 → 월별 타임머신으로

연혁에 박힌 사건(예: "2020년 대학교 입학")을 **클릭하면**:
- **앵커(정확)** → 그 시기의 월별 타임머신(/timemachine/2020/3)으로 이동 → 시대 사건·음악·AI 비서로 회상하며 회고 작성.
- **사이(대략)** → 추정 연도의 월별 타임머신으로 이동 (정확한 월 없으면 그 연도의 대표 월 — 예: 3월).

v2에서 만든 모든 것(시대 사건 DB, 음악 차트, AI 비서, 음성→AI 다듬기, 가족 반응 등)이 *다른 용도*로 그대로 살아남:
- 메인 입구는 연혁(v3) → 큰 줄기 잡기
- 깊이 가고 싶을 때 월별(v2) → 시대 회상하며 글 쓰기

---

## 6. AI 비서 — 우측 상단 버튼

- 평소엔 닫혀있음.
- 우측 상단 "AI 비서와 대화" 버튼 → 누르면 기존 채팅 UI(V3, 대화형+저장+3단계 깊이) 띄움.
- **초기 질문 흐름(섹션 3)과 별개** — 초기 질문은 *가이드형 폼*이고, 비서는 *자유 대화형*.
- 사용자가 연혁을 보다 "이때 무슨 일 있었지?" 떠올리고 싶으면 비서 호출해 대화.
- 토큰 구조 그대로(V4 3단계 깊이, 답 저장, 출처).

---

## 7. 월별 타임머신(v2)은 어디로?

- **메인에서 빠짐.** 사용자가 직접 /timemachine/[year]/[month]로 들어가거나, 연혁의 사건 클릭으로만 진입.
- **모든 코드 그대로 유지** — 데이터·UI·저장·가족 반응·진척 시각화 다 살아있음. *진입 동선*만 바뀜.
- 사이드 패널·출석체크는 v3에서도 그대로 노출 (메인은 연혁이 됨).

## 8. 기존 구현(v2, T1~T6, 출석, 비서 V1~V4, 진척, 가족 반응)과의 관계

### 살린다 (거의 다)
- **MonthEvent, ChartSong, UserMemory, MemoryReaction, FamilyFeedSeen, UserAttendance** — DB 모델 다 그대로.
- **AI 비서(V1~V4)** — 우측 상단 버튼으로 호출. 기능 그대로.
- **월별 타임머신 화면** — 보조로 유지, 코드 무변경.
- **출석체크, 사이드 패널, 진척(①), 가족 반응(②)** — 다 그대로.
- **데이터(1년치 시드)** — 그대로.

### 추가
- **인생 연혁 화면 (가로 시간축)** — 새 메인.
- **초기 질문 폼** — 인생 골격 잡는 가이드.
- **인생 이벤트(life_event) 데이터** — UserMemory에 얹기.

### 바꾼다
- **/timemachine 메인 페이지** — 출석 카드 + 진척 + 가족 소식 → 그대로 두되, **연혁이 화면 상단의 메인**이 됨. 출석·진척·가족 소식은 *연혁 아래*로.
- **기본 진입** — 로그인 후 첫 진입은 연혁. 처음이면 "인생 기록하기" 시작.

### 안 지움
- 월별 타임머신 화면 코드, 음성, 음악 카드, 비서, 출석, 진척, 가족 반응 — 다 보존.

## 9. 구현 Phase 제안

- **L1. 데이터 모델** — UserMemory에 인생 이벤트 필드 추가(createdVia="life_event", eventTitle, eventYear, eventMonth?, precision, category). 마이그레이션.
- **L2. 초기 질문 폼** — "인생 기록하기" 시작 → 정해진 질문 차례로 → 답하면 life_event UserMemory에 저장. 건너뛰기·중간 저장 지원.
- **L3. 인생 연혁 화면** — 가로 시간축 시각화. 앵커(진한 큰 점) + 사이(작은 점/점선). 클릭 → 해당 월별 타임머신으로 이동.
- **L4. 인생 이벤트 추가** — 연혁에서 "+ 인생의 한 장면 추가하기" 버튼 → AI 대화로 추정 연도·앵커 기반 사이 위치 결정 → 저장.
- **L5. 메인 페이지 재배치** — /timemachine의 메인을 연혁 중심으로. 출석·진척·가족 소식은 아래로.
- **L6. AI 비서 호출 버튼** — 우측 상단, 평소 닫힘, 누르면 기존 채팅 띄움.
- **L7. 첫 진입 흐름** — 로그인 후 인생 이벤트 0건이면 "인생 기록하기" 권유, 있으면 연혁 바로.

## 10. 핵심 검증 질문 (구현 후)

**"매달 채우는 부담이 사라지고, 사용자가 *자기 인생 한 줄기*를 한 번 그어보고 싶어지는가?"**

특히:
- 초기 질문 흐름이 *지치지 않게* 끝나는가? (10~20분 이상 걸리면 부담)
- 사이 이벤트(대략 시점) 추가가 *정확한 월 안 떠올려도* 작동하는가?
- 연혁이 한눈에 *내 인생*으로 보이는가?
- 깊이 파고 싶을 때 월별로 가는 게 자연스러운가?

## 11. 열린 질문 (구현 중/후 결정)
- 초기 질문의 정확한 개수·순서·문구 (지나치게 길지 않게)
- "건너뛰기" 후 나중에 채울 수 있는 동선
- 가족 룸에서 인생 연혁 공유 형태 (전체 연혁? 개별 이벤트?)
- 사이 이벤트의 시각적 표현 디테일 (점 크기·색·위치 알고리즘)
- 모바일에서 가로 시간축 처리 (가로 스크롤? 세로 변환?)
