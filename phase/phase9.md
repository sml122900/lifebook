# Phase 9 — 가족 공유 (개인 공유 + 공동 추억)

> **목표**: 가족·커플이 한 룸에 모여 서로의 추억을 **함께 보고 댓글**로 떠들고, 더 나아가 **공동 추억**(우리가 함께 겪은 일)을 같이 채운다. 제품의 *원래 목적* — "부모님·배우자와 함께 떠드는 도구" — 을 완성하는 단계.
> **선행 조건**: MVP(Phase 0~8) 완료, `git tag mvp-v1` 권장.
> **분량**: 큼. 권한·프라이버시가 가장 복잡 → 단계를 끊어 진행.

**핵심 데이터 구분 (Phase 9의 뼈대)**
- **개인 추억(UserMemory)**: 작성자 1명 소유. 룸에서 보고 댓글 가능하나 **수정·삭제는 작성자만.**
- **공동 추억(SharedMemory)**: 룸 공동 소유. 룸 멤버 누구나 함께 작성·기여. "우리 것."

**불변 원칙 (CLAUDE.md)**
- 솔로=비공개 기본. **공유는 명시 동의 + 범위(룸) 한정.**
- 룸 초대받은 멤버만 접근. 모든 조회에 **룸 멤버십 검증** (userId 스코프 → roomId+멤버십 스코프로 확장).
- ⚠️ **권한 변경/문서 공유 행위는 사용자 본인이** — Claude Code는 권한 로직을 *구현*하되, 실행 중 권한·공유 설정을 임의로 바꾸지 않는다.
- 약관에 "타인 정보 입력 책임은 사용자에게" 보증 조항(법무 검토 필요).

**진행 순서**: A(룸+개인 공유)를 먼저 완성·검증 → B(공동 추억)를 얹는다.

# 트랙 A — 룸 + 개인 추억 공유

## 9.1 — 룸 모델 + 생성

**작업**
- `SharedRoom`(`id, name, ownerId, createdAt`), `RoomMember`(`roomId, userId, role: owner|member, consentAt, joinedAt`, `@@unique([roomId, userId])`) 모델 + 마이그레이션 + generate + 재시작.
- 룸 생성 UI(이름 예: "우리 가족", "엄마와 나").

**완료 기준**: 룸을 만들면 생성자가 owner 멤버로 들어간다.

## 9.2 — 초대 + 동의 게이트

**작업**
- 초대: 초대 링크/코드 발행(이메일 발송은 후순위, 링크 공유로 시작).
- 초대받은 사람이 링크로 들어오면 **공유 동의 게이트**: "이 룸에 참여하면 내 추억이 룸 멤버에게 보입니다" 명시 동의 → `consentAt` 기록 후 합류.
- ⚠️ 사전 체크·자동 동의 없음. 동의 안 하면 합류 안 됨.

**완료 기준**: 초대 링크로 들어와 명시 동의 후 멤버로 합류, 동의 시각 저장.

## 9.3 — 공유 타임라인 (함께 보기)

**작업**
- 룸 화면: 멤버들의 **개인 추억을 합산**해 한 타임라인에 표시(작성자 표시).
- ⚠️ **멤버십 검증**: 룸 멤버가 아니면 접근 불가. 조회는 roomId + 멤버십으로 스코프.
- 개인 추억은 룸에서 **읽기 전용**(작성자 본인만 자기 화면에서 수정).
- 시니어 접근성 유지.

**완료 기준**: 두 계정이 한 룸에서 서로의 추억을 함께 본다(타인 추억 수정 불가).

## 9.4 — 댓글 ("떠들기")

**작업**
- 추억(개인/공동)에 댓글 모델(`Comment`: `id, roomId, targetType, targetId, authorId, content, createdAt`) + 작성/조회.
- 룸 멤버만 작성·조회. 본인 댓글만 삭제.
- 따뜻한 톤의 빈 상태("이 추억에 대해 한마디 남겨보세요").

**완료 기준**: 멤버가 추억에 댓글을 달고 서로 본다.

**▶ 트랙 A 체크포인트**: 룸 생성 → 초대 → 동의 → 함께 보기 → 댓글이 두 계정으로 동작. 여기서 멈춰 검증 후 트랙 B.

# 트랙 B — 공동 추억

## 9.5 — 공동 추억 모델

**작업**
- `SharedMemory`(`id, roomId, year, month, title, content, createdById, createdAt`) — **룸 소유**. 
- (선택) 기여자 추적: `SharedMemoryEdit` 또는 `lastEditedBy`.
- 마이그레이션 + generate + 재시작.

**완료 기준**: 룸에 속한 공동 추억을 저장할 구조가 생긴다.

## 9.6 — 공동 추억 작성 (함께 채우기)

**작업**
- 룸 멤버 누구나 공동 추억 작성/편집("우리가 함께 겪은 일"). 예: "신혼여행 — 1998", "첫째 태어난 날".
- 권한: 룸 멤버면 작성·편집 가능(공동 소유). 삭제는 작성자 또는 룸 owner.
- (선택) 여기에도 AI 가이드 질문을 붙일 수 있으나, **토큰 차감 정책 동일 적용**(누구 지갑에서 차감할지 명확히 — 작성 시도한 멤버 본인).

**완료 기준**: 멤버들이 하나의 공동 추억을 함께 만들고 편집한다.

## 9.7 — 공유 타임라인에 공동 추억 통합

**작업**
- 공유 타임라인에 [세상 사건] + [개인 추억(작성자별)] + [공동 추억] 을 함께 표시, 시각적으로 구분.
- 공동 추억은 "우리" 라벨/색으로 개인 추억과 구분.

**완료 기준**: 한 룸 타임라인에서 세 종류가 어우러져 보인다.
