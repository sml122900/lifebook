# Phase 5 — 타임라인 개인화 (출생연도 기반)

> **목표**: Phase 2의 하드코딩을 걷어내고, 로그인 사용자의 **출생연도**에 맞춰 타임라인 시대 범위를 개인화한다. "그때 나는 몇 살이었지?"가 보이게 한다.
> **선행 조건**: Phase 4 완료(온보딩에서 birthYear 수집), Phase 3 인증 동작.
> **분량**: 짧다. 사실상 birthYear를 타임라인에 연결하는 작업.
> **작업 방식**: 5.1부터 하나씩, 완료 기준 충족 시 커밋 후 다음.

**비범위**: 트리거 이벤트는 Phase 6. 여기서는 **앵커 이벤트만** 다룬다.

---

## 5.1 — birthYear 연결 (하드코딩 제거)

**목적**: 임시 상수 대신 실제 사용자 데이터를 쓴다.

**작업**
- Phase 2의 `DEMO_BIRTH_YEAR` 제거.
- `/timeline` 서버 컴포넌트에서 현재 세션 사용자의 `User.birthYear`를 조회해 사용.

**완료 기준**: 로그인 사용자의 실제 출생연도가 타임라인 로직에 들어간다.

---

## 5.2 — 시대 범위 필터

**목적**: 사용자가 "겪은 시대"를 중심으로 보여준다.

**작업**
- 기본 범위: **birthYear ~ 현재 연도**의 앵커 이벤트.
- (선택) 출생 직전 몇 년(예: 부모 세대 사건)은 흐리게/접힌 형태로 두는 옵션 — 원하면.
- 쿼리: `where: { category: "anchor", year: { gte: birthYear } }`.

**완료 기준**: 사용자 출생연도 이후의 앵커 이벤트만(또는 그 범위 위주로) 보인다.

---

## 5.3 — "그때 나는 N살" 나이 표기

**목적**: 회고의 핵심 트리거. 사건 옆에 당시 나이를 보여주면 기억이 훨씬 잘 떠오른다.

**작업**
- 각 연도/사건에 `해당연도 - birthYear` 로 나이 계산해 작게 표기(예: "그때 14살").
- birthYear가 없으면 표기 생략(5.5 참고).

**완료 기준**: 각 연도에 사용자의 당시 나이가 자연스럽게 표시된다.

---

## 5.4 — "내 사건" 레이어를 userId 스코프로 연결

**목적**: Phase 7에서 추억 카드가 들어올 자리를, 지금 **안전한 쿼리**로 미리 연결한다.

**작업**
- "내 사건" 레이어에 `UserMemory`를 조회해 표시(현재는 데이터 없어 빈 상태 유지).
- ⚠️ **반드시 `where: { userId: 현재유저 }`로 스코프** — 타인 데이터가 절대 섞이면 안 됨. (Phase 3에서 약속한 솔로=비공개 원칙의 첫 실제 쿼리)

**완료 기준**: 내 사건 쿼리가 현재 사용자로 스코프되어 있고(코드 확인), 데이터 없을 때 빈 상태가 그대로 보인다.

---

## 5.5 — 엣지 케이스 (birthYear 없음)

**목적**: 온보딩을 건너뛴 사용자 대응.

**작업**
- birthYear가 없으면: 전체 앵커 이벤트를 보여주되, "출생연도를 입력하면 더 맞춤형으로 볼 수 있어요" 같은 가벼운 안내 + 입력 링크(온보딩/설정).
- 나이 표기는 생략.

**완료 기준**: birthYear 없는 사용자도 타임라인이 깨지지 않고, 입력을 부드럽게 권유받는다.
