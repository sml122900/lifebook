# Lifebook "타임머신" 구현 기획서 (Phase 분할)

> **목적**: Claude Code에게 단계별로 지시하기 위한 구현 기획서.
> **전제**: `타임머신_리디자인_기획.md`의 방향을 확정하고, 구현 가능한 Phase로 분할한 것.
> **검증 범위**: 최근 1년(2025년 6월 ~ 2026년 5월)만 먼저 만들어 검증. 반응 좋으면 과거로 확장.
> **기존 코드와의 관계**: Phase 5(출생연도 타임라인)·6(음악 RAG)을 대체. Phase 7(추억 저장·음성·RAG 가드)·9(공유·결제·동의)는 유지.

| 항목 | 결정 |
|---|---|
| 진행 방향 | 현재(2026.5)부터 **한 달씩 거꾸로** |
| 화면 구성 | "○년 ○월엔 이런 일이 있었어요" + 4섹션 사건 + 음악 |

| 검증 범위 | 2025.6~2026.5 (12개월) 먼저 |

## Phase T1 — 데이터 모델 (월·사건·유행·음악)

**목표**: 타임머신의 모든 콘텐츠를 담을 스키마 정의 + 마이그레이션.

**할 일**
1. `MonthEvent` 모델 — 월별 사건
   - `id`, `year`(Int), `month`(Int 1~12)
   - `section`(enum: POLITICS_SOCIETY / CULTURE / SPORTS / TREND)
   - `title`(String, 사건명), `description`(String?, 짧은 설명)
   - `eventDate`(String?, "2025-06-14"처럼 알려진 경우만. 유행은 null)
   - `source`(String?, 출처), `confidence`(enum: VERIFIED / APPROX) — 검증 표시
2. `TrendPeriod` 처리 — 유행은 `MonthEvent`에 합치되, **기간 노출**을 위해:
   - `startYear`,`startMonth`,`endYear`,`endMonth` (유행/해외음악의 노출 기간)
   - 일반 사건은 이 필드 null, 단일 월(`year`/`month`)로 노출
   - 유행·해외음악은 기간 필드 사용 → 쿼리 시 "해당 월이 기간에 포함되면 노출"
3. `ChartSong` 모델 — 음악
   - `id`, `rank`(Int?, 국내는 1~10 / 해외는 null 가능)
   - `title`, `artist`
   - `origin`(enum: DOMESTIC / INTERNATIONAL)
   - 국내: `year`,`month` (월 단위 정밀)
   - 해외: `startYear`,`startMonth`,`endYear`,`endMonth` (기간 노출)
   - `youtubeQuery`(String, 검색 링크용 "곡명 아티스트")
   - `eraColor`(String?, 시대의 색 키 — 디자인용)
4. 마이그레이션 생성 + 적용.

**완료 기준**: 위 모델로 "특정 연·월을 주면 그달에 노출할 사건·유행·국내음악·해외음악을 모두 가져오는" 쿼리가 가능.

**주의**: 기간 노출 쿼리 = `(startY*12+startM) <= (targetY*12+targetM) <= (endY*12+endM)`. 월을 정수로 환산해 비교하면 간단.

## Phase T2 — 데이터 시드 (1년치)

**목표**: 2025.6~2026.5의 사건·유행·음악 데이터를 DB에 시드.

**데이터 소스 (확정)**
- **사건 4섹션**: 어시스턴트(Claude)가 웹 검색으로 아카이빙 → 엑셀 → 사용자 검토·수정 → JSON/시드 변환.
- **유행**: 위와 함께. 노출 기간(시작월~끝월) 포함.
- **국내 음악**: 사용자가 멜론 월간 1~10위를 직접 수작업 입력.
- **해외 음악**: 어시스턴트가 빌보드 기반 "그 무렵 대표곡" + 노출 기간 아카이빙.

**워크플로**
1. 어시스턴트가 사건·유행·해외음악을 검색·검증 → **엑셀 파일**로 제출(섹션·기간·출처·신뢰도 컬럼).
2. 사용자가 엑셀 검토·수정·국내음악 추가.
3. 확정 엑셀 → `seed-timemachine.ts` 로 변환 → DB 시드.

**완료 기준**: 12개월 각각에 대해 4섹션 사건 + 유행 + 국내10곡 + 해외 대표곡이 채워짐. 빈 달 없음.

**저작권 가드**
- 사건: 사실만, 출처 서술 문장 복제 금지.
- 음악: 순위·곡명·아티스트(사실)만. 멜론/빌보드 표·디자인 복제 금지. 앨범/가수 이미지 미사용.

## Phase T3 — 월 화면 UI (핵심 경험)

**목표**: "한 달"을 보여주는 핵심 화면.

**할 일**
1. 라우트: `/timemachine` (기본 진입 = 2026년 5월).
2. 헤더: "○○○○년 ○월" 크게 + "이런 일이 있었어요. ○○님에겐 어떤 일이 있었나요?" 멘트.
3. 4섹션 사건 목록(정치·사회/문화/스포츠/유행). 각 항목:
   - 사건명 + 짧은 설명
   - **체크(남김) / 삭제(X)** 토글
   - 남긴 항목 옆 **내 이야기 입력란** (텍스트)
4. 음악 섹션: 국내 1~10위 + 해외 대표곡. (카드 디자인은 T5)
5. 내비게이션:
   - **"← 이전 달"** 버튼(거꾸로)
   - **건너뛰기** 버튼(이 달 그냥 넘김) — 필수
   - 빠르게 여러 달 넘기기 가능(연속 클릭이 매끄럽게)
   - ※ 시대 점프(바로가기)는 만들지 않음

**완료 기준**: 2026.5에서 시작해 한 달씩 거꾸로 이동하며, 각 달의 사건·음악이 뜨고, 체크/삭제/내이야기 입력이 동작.

## Phase T4 — 음성 입력 → AI 다듬기

**목표**: 내 이야기를 음성으로 녹음하면 AI가 문장으로 다듬어 입력.

**할 일**
1. 사건 옆 입력란에 **마이크 버튼**(시니어 친화: 크게, 상태 명확).
2. 녹음 → STT → AI가 **사용자가 말한 사실만** 문장으로 정리(창작·추가 금지 = RAG 가드 동일 원칙).
3. 결과를 입력란에 채우고, 사용자가 수정 가능.
4. 토큰 차감(기존 결제 구조 연동) — 추억 저장 단위로.

**완료 기준**: 음성으로 말한 내용이 다듬어진 텍스트로 입력되고, 사용자가 검토·수정·저장 가능.

**주의**: Phase 7의 음성·RAG 가드 로직 재사용. AI는 사용자 발화를 벗어난 내용을 넣지 않는다.

## Phase T5 — 음악 카드 디자인 (이미지 없이)

**목표**: 음악 섹션을 감성적으로. 이미지 0, 저작권 0.

**할 일**
1. 카드 요소: **순위 원형 배지**(1위 강조색) + 곡명/아티스트 + **재생 아이콘(▶)**.
2. **시대의 색** 적용: `eraColor` 키에 따라 카드 색감 변화(예: 80s 세피아, 90s 네온, 00s 메탈릭, 2010s~ 미니멀).
3. 1위 크게/강조, 하위는 작게(타이포 위계).
4. 재생 아이콘/제목 클릭 → **유튜브 검색 결과 링크**(새 탭). `youtubeQuery` 사용.
5. (선택) LP·카세트 모티브 아이콘(오픈 라이선스/직접 SVG).

**완료 기준**: 국내·해외 음악이 시대의 색이 입혀진 카드로 표시되고, 클릭 시 유튜브 검색으로 이동.

**금지**: 앨범 커버·가수 사진 사용 금지(저작권·초상권). 임베드 재생 금지.

## Phase T6 — 기존 구조 연결 & 정리

**목표**: 타임머신을 기존 추억·공유·결제와 통합하고, 대체된 Phase 정리.

**할 일**
1. 타임머신에서 입력한 "내 이야기"를 기존 `UserMemory`(추억)로 저장 → Phase 9 공유·책 제작과 연결.
2. 기존 Phase 5(출생연도 타임라인)·6(음악 RAG) 화면을 타임머신으로 대체(또는 비활성).
3. 온보딩 흐름에서 타임머신으로 자연스럽게 진입하도록 연결.
4. 사용 안 하게 된 RAG·임베딩·세대가중 로직 정리(제거 또는 보류 표시).

**완료 기준**: 타임머신에서 채운 추억이 책·공유로 이어지고, 결제(토큰)·동의 구조가 그대로 작동.

## 검증 계획 (1년치로 무엇을 확인하나)

핵심 질문: **"사용자가 한 달 한 달 넘기며 계속 채우고 싶어 하는가?"**
- 막히지 않고 즐겁게 넘기는가
- 사건 나열 → 체크/삭제 → 내 이야기 흐름이 자연스러운가
- 음악이 회상을 돕는가
- 건너뛰기·빠른 이동이 충분한가

→ 이게 되면 과거로 확장(시대별 데이터 시드). 안 되면 흐름 재설계.
