### #9 STEP 2 — Google Places Autocomplete 구현

#### 배경 + 인코딩 버그 발견·수정
- 이전 세션에서 "중산고" 0건, "강남역" 0건 → 구글 한국어 문제로 오판
- 이성민 지적: 브라우저 직접 검색은 다 뜸 → 요청 인코딩 버그 의심
- 진단: curl/heredoc → Windows CP949 인코딩으로 전송 → 구글 서버가 UTF-8 해석 시 키릴 문자(Б) 오매핑(mojibake)
- 수정: Python `json.dumps(ensure_ascii=False).encode('utf-8')` + `urllib.request`로 테스트
- 결과: "중산고" 5건·"여의도고" 5건·약칭 전부 정확 → API 정상

#### 백엔드 (`app/api/place-search/route.ts`)
- `autocompleteGoogle(input)` 신규
  - `places:autocomplete` POST, `Content-Type: application/json; charset=utf-8` 명시 (mojibake 방지)
  - body: `{ input, languageCode:"ko", regionCode:"KR", locationBias: 서울 100km circle }`
  - 응답 `suggestions[].placePrediction` → `{ text, placeId }[]`
- `getGooglePlaceDetail(placeId)` 신규
  - `GET places/{placeId}?languageCode=ko`, FieldMask: `id,displayName,formattedAddress,location`
  - 반환: `PlaceResult` (name/address/lat/lng)
- `parseBody()` 확장
  - `action: "autocomplete" | "detail" | "search"` 3-way 분기
  - placeId path injection 방지: `/^[A-Za-z0-9_-]{5,200}$/` 검증
- 기존 네이버·구글 searchText 경로 **무수정**

#### 프론트 (`app/components/PlaceSearchInput.tsx`)
- `DEBOUNCE_MS` 500 → 300
- `Suggestion` 타입 (`{ text, placeId }`) + `suggestions` / `detailLoading` 상태 추가
- `useEffect` 내 엔진 분기
  - Google: `action:"autocomplete"` → `suggestions` 세팅
  - Naver: 기존 text search → `results` 세팅 (무변)
- `pickSuggestion(s)` — 후보 선택 시 `action:"detail"` 조회 → `pick()` 호출
- B단계 렌더 분리
  - Google: autocomplete 드롭다운 (56px 행, 좌표 없어 지도 안 보임)
  - Naver: 기존 결과 목록 + 지도 타일 (무변)
- 입력 placeholder 갱신: "예: 강남역, 중산고, 도쿄역"
- Google C단계(선택 완료): 기존 GoogleMap 타일 그대로 표시

### #10 STEP 0 — CLOVA Speech 조사 + 포맷 테스트

#### 조사 (코드 변경 없음)
- CLOVA Speech (New) 장문 인식: REST, 비동기 (POST job → polling GET)
- Phase 1 권장: sync 모드(`"completion":"sync"`) → 짧은 통 녹음에서 폴링 없이 결과 즉시 수신
- 엔드포인트: `POST {invokeUrl}/recognizer/upload` (multipart/form-data)
- 필요 키: `CLOVA_SPEECH_SECRET` + `CLOVA_SPEECH_INVOKE_URL`
- 재사용 가능: `uploadRecording()`, `/api/recordings`, `UserMemory.audioPath`
- 신규 필요: `FreeRecorder` 컴포넌트, `lib/clova-speech.ts`, `/api/clova-stt` 라우트

#### 포맷 테스트 결과 (로컬 실행)

| 포맷 | Content-Type | 결과 |
|---|---|---|
| WAV | audio/wav | 200 COMPLETED ✓ |
| OGG | audio/ogg | 200 COMPLETED ✓ |
| WebM | audio/webm | 200 COMPLETED ✓ |
